# 算法 · 背包动态规划（Knapsack DP）

## 导读

**背包问题**是动态规划中最具辨识度的模型之一：给定若干物品，每件有重量（或成本）与价值，在总容量不超过上限 `W` 的前提下，求能装入背包的最大价值。名称来自「在容量约束下做选择」的隐喻；与线性 DP 沿时间或前缀推进不同，背包的状态往往同时依赖**已考虑到的物品前缀**与**当前剩余容量**——在实现上我们常把二维 `dp[i][w]` 压成一维 `dp[w]`，用遍历顺序区分「每件最多选一次」与「每件可重复选」。

Study 仓库 `knapsack/` 提供两个核心函数，覆盖面试与竞赛的八成场景：

| 函数 | 模型 | 每件物品 |
|------|------|----------|
| `knapsack_01` | 0-1 背包 | 最多选 1 次 |
| `knapsack_unbounded` | 完全背包 | 可重复选无限次 |

本页在 `notes.md`「0-1 / 完全背包」骨架上扩写：状态定义、二维到一维的空间优化、**逆序与正序**容量循环为何不能写反、与 **322 零钱**、**416 分割等和** 等 LeetCode 题的映射，以及和 **分数背包（贪心）** 的本质区别——后者在物品可分割时用价值密度排序，**不能**用 0-1 DP 一维数组替代；对比阅读站点专题 `algo-greedy`（分数背包与活动选择等贪心模板）。

若你刚学完 `algo-dp-linear`，可把背包看作「多了一个容量维度」的选/不选：0-1 背包的转移与打家劫舍类似，都是「当前决策影响未来能否再选」，但背包的「冲突」体现在重量占用而非相邻下标。读完本文，你应能默写 Study 两个函数、解释 `for x in range(cap, w-1, -1)` 的必要性、将题面改写成 `dp[w]` 语义，并在 Python 与 C++ 中对拍断言。

**与贪心的边界**：当题目允许把物品**切开**取一部分重量时，最优策略是按 `value/weight` 从高到低装入——这是 `O(n log n)` 排序贪心，见 `algo-greedy` 中分数背包一节。0-1 与完全背包要求**整件**选取，离散选择导致组合爆炸，才需要 `O(nW)` DP。面试时先问清「能否分割」，再选算法族。

**在 DP 家族中的位置**：`algorithms/dynamic_programming/knapsack` 与 `linear`、`interval`、`tree` 等并列；总览见 `algo-dynamic-programming`。Hot 100 与面试高频里，背包常与「子集和」「划分」「硬币」组合出现，本页聚焦**容量维滚动**这一实现核心，多重背包、分组背包、依赖背包等变体在基础篇「变体与技巧」中点到为止，细节以 Study 题解目录延伸。

**为何值得系统学**：背包 DP 的错误模式高度集中——循环方向写反、初始化语义搞错、把分数背包当 0-1 建模——修一处往往全局通过。掌握 Study 两个函数后，你拥有可运行的「锚点」，变体题只是在同一 `dp` 数组上改初始化、改转移（`max` 改 `min`、赋值改 `+=`、布尔改计数）。这与线性 DP 四模板同一 pedagogy：先正确实现仓库代码，再练题面改写。

**面试沟通顺序**：30 秒定义 `dp[x]` 语义 → 说明 0-1 还是完全 → 写出双重循环方向 → 报 `O(nW)` 与空间 `O(W)` → 提及分数背包时转 `algo-greedy`。写代码前不要跳过方向说明，考官常在此处追问「为何逆序」。

> **环境**：Python 3.10+；C++17，`g++` 编译，Study 侧 `knapsack_dp.cpp` 通过 `#include <alg_std.hpp>` 使用 `vector`、`max` 等。

阅读本专题前，建议已具备：

- **动态规划基础**：最优子结构、重叠子问题、填表与滚动数组；若生疏可先读 `algo-dp-linear` 导读中的「无后效性」表述。
- **数组与循环**：理解嵌套循环 `for 物品` × `for 容量` 的执行顺序如何决定「每件用一次还是多次」。
- **复杂度记号**：`O(nW)` 中 `W` 为容量上界（伪多项式）；`n` 为物品个数。`W` 达 `10⁶` 时一维 DP 可能 TLE，需另寻方法（单调队列优化、二进制拆分等）。
- **整数 overflow**：价值与重量累加在竞赛中常用 `long long`；Python 整数任意精度，C++ 实现需注意。

**0-1 与完全的口诀（先记住再证明）**：

- 0-1：外层枚举物品，内层容量 **从大到小**（`cap` → `w`），保证同一物品本轮只更新一次。
- 完全：外层枚举容量 **从小到大**（`1` → `cap`），内层枚举物品；或外层物品、内层容量 **从小到大**，等价于允许同一物品在同一轮贡献多次。

**伪多项式**：背包 DP 的时间依赖 `W` 的数值而非 `log W`，因此不属于对输入规模的多项式算法；面试中仍默认 `n·W` 可接受（如 `n=100, W=10000`）。若 `W` 极大而 `n` 小，考虑 meet-in-the-middle 或特殊结构，已超出本专题默认范围。

**状态设计习惯**：先固定「`dp[x]` 表示什么」再写循环。常见三种：(1) 容量不超过 `x` 的最大价值；(2) 恰好容量 `x` 是否可达 / 方案数；(3) 凑出金额 `x` 的最少硬币数。三种的初始化与答案下标不同，不可混用「全零 + 返回 `dp[cap]`」于所有题。

**工具链**：本地双终端对拍 Python 与 C++；改循环方向后务必重跑 `knapsack_dp.py` 与 `run.exe`。Windows 下路径含空格时用 `-LiteralPath`；见 Study 仓库对照一节。

## 基础篇

### 直觉与定义

**问题抽象**

有 `n` 件物品，第 `i` 件重量 `w[i]`、价值 `v[i]`，背包容量上限 `cap`（非负整数）。求在不超过容量的前提下，能取得的最大总价值。

**0-1 背包（0/1 Knapsack）**

每件物品**要么不选，要么选一整件**，不能重复。子问题可定义为：

- **二维**：`dp[i][x]` = 只考虑前 `i` 件物品、容量恰好不超过 `x` 时的最大价值（或「容量为 `x`」时的最大价值，两种写法需固定语义）。
- **转移**：对第 `i` 件，不选则 `dp[i][x] = dp[i-1][x]`；选则要求 `x >= w[i]`，`dp[i][x] = max(dp[i][x], dp[i-1][x-w[i]] + v[i])`。
- **边界**：`dp[0][x] = 0`；答案 `dp[n][cap]`。

**一维压缩**：`dp[x]` 表示当前已处理若干物品后，容量为 `x` 的最大价值。处理新物品 `(w, v)` 时，必须从 `x = cap` 递减到 `x = w`，令 `dp[x] = max(dp[x], dp[x-w] + v)`。递减保证 `dp[x-w]` 仍是「未放入当前物品」的状态；若从小到大，同一物品会在一轮内被用多次，退化成完全背包。

**手推样例（Study 断言）**

物品 `(w,v)`：`(1,6), (2,10), (3,12)`，容量 `5`。最优选重量 2+3=5，价值 10+12=**22**。填表时第三件更新 `dp[5]` 时用到 `dp[2]=10`。

**完全背包（Unbounded Knapsack）**

每种物品**无限供应**。子问题 `dp[x]` = 容量不超过 `x` 的最大价值（或恰好 `x`，见变体）。转移：

`dp[x] = max over items ( dp[x-w] + v )`，其中 `x >= w`。

一维实现：外层 `x` 从 `1` 到 `cap`，内层枚举物品——等价于「每种物品可重复贡献」。Study 的 `knapsack_unbounded` 采用该顺序；样例 `w=[1,3,4], v=[15,50,60], cap=8` 最优为两个 4 重量（60+60=120）或 1+3+4 组合，断言结果为 **130**（三个 1 重量 15×3 加 3 重量 50 等组合需手算验证代码：8=1+1+1+1+1+1+1+1 价值 105；8=4+4 价值 120；8=3+1+4 价值 125；8=1×2+3×2 等，实际最优 130 来自 1×2 + 3×2 = 15×2+50×2=130）。

**与分数背包对比**

若物品可分割，按 `v/w` 降序贪心装入直至满，复杂度 `O(n log n)`，最优性由交换论证保证。**0-1 与完全背包不允许分割**，贪心密度排序可能极差（例如一大件价值略低于两件小件合计）。站点 `algo-greedy` 专讲可分割情形；本题专讲离散 DP。面试一句话：**能切用贪心，不能切用背包 DP**。

**二维语义图示**

想象表格行是物品前缀、列是容量 `0..cap`。0-1 每行只从上一行拷贝或「上一行左侧一格加价值」跳一步；完全背包同一行可从**本行**左侧转移，故压缩后需正序容量。初学者可先写 `dp = [[0]*(cap+1) for _ in range(n+1)]` 再改一维，减少方向错误。

**「恰好装满」与「至多装满」**

Study 函数求的是**容量不超过 `cap` 的最大价值**（在不超过的意义下，一维 `dp[x]` 常表示「恰好容量为 x」可达的最大价值，最终答案取 `dp[cap]` 时，若定义是「恰好」则须保证只有装满 `cap` 才更新答案；LeetCode 494 等「恰好和」用 `dp[0]=1` 计数或 `dp[target]` 判定可达）。题面关键词「恰好」「装满」「等于」要改初始化：不可达状态设为 `-inf` 或 `False`，而非全零最大价值。

**空间优化：从二维到一维**

原始 `dp[i][x]` 占 `O(n·cap)`。观察第 `i` 行只依赖第 `i-1` 行，可删去行下标，保留 `dp[x]` 并在外层枚举物品。0-1 压缩后**必须逆序** `x`，完全背包压缩后**必须正序** `x`——这是本专题与 `algo-dp-linear` 中 LCS 一维滚动「内层 j 从大到小」同构的「方向敏感」现象：压缩维度后，循环顺序承担了「保留旧层语义」的职责。初学者可先用 5×5 小表手填二维两行，再对照一维数组在处理一件物品时的变化，亲眼看到正序时同一行被多次使用。

**记忆化写法（面试口述用）**

```python
def knapsack_01_memo(weights, values, cap):
    n = len(weights)
    from functools import lru_cache
    @lru_cache(maxsize=None)
    def f(i, rem):
        if i == n or rem == 0:
            return 0
        best = f(i + 1, rem)
        if weights[i] <= rem:
            best = max(best, values[i] + f(i + 1, rem - weights[i]))
        return best
    return f(0, cap)
```

迭代一维写法常数更小、无递归深度问题；记忆化帮助证明最优子结构。状态 `(i, rem)` 共 `(n+1)(cap+1)` 个，与填表同阶。

### 复杂度分析

| 模型 | 状态规模 | 时间 | 空间（一维） |
|------|----------|------|----------------|
| 0-1 背包 | `n × cap` | `O(n · cap)` | `O(cap)` |
| 完全背包 | `n × cap` | `O(n · cap)` | `O(cap)` |
| 二维未压缩 | `(n+1)(cap+1)` | `O(n · cap)` | `O(n · cap)` |

瓶颈在 **`cap` 的数值**：`cap=10⁵, n=10²` 约 `10⁷` 运算，C++ 通常可过，Python 需注意常数。若 `n` 大且 `cap` 大，需优化（二进制拆分多重背包、单调队列优化完全背包等），本页实现为教科书基准版。

**与暴力对比**：每件选或不选为 `O(2^n)`；DP 将「前 i 件 + 容量 x」合并状态，避免重复子问题。

**空间优化小结**：二维可滚为一维；0-1 **逆序**容量；完全 **正序**容量。不要误以为「背包都能滚成一维且方向任意」——方向错误是 WA 首要原因。

**价值维度对调**：若重量连续、价值离散且范围小，可交换「重量/价值」角色做 DP（罕见）；默认仍以容量维为主。

### 代码模板

以下与 Study 仓库逻辑一致，便于默写。

**0-1 背包 — 一维逆序**

```python
def knapsack_01(weights: list[int], values: list[int], cap: int) -> int:
    dp = [0] * (cap + 1)
    for w, v in zip(weights, values):
        for x in range(cap, w - 1, -1):
            dp[x] = max(dp[x], dp[x - w] + v)
    return dp[cap]
```

**完全背包 — 容量正序 + 物品**

```python
def knapsack_unbounded(weights: list[int], values: list[int], cap: int) -> int:
    dp = [0] * (cap + 1)
    for x in range(1, cap + 1):
        for w, v in zip(weights, values):
            if x >= w:
                dp[x] = max(dp[x], dp[x - w] + v)
    return dp[cap]
```

**等价写法（完全背包，外层物品、内层正序）**

```python
for w, v in zip(weights, values):
    for x in range(w, cap + 1):
        dp[x] = max(dp[x], dp[x - w] + v)
```

两种完全背包写法均合法；Study 采用第一种（先容量后物品），与部分教材「硬币」写法一致。

**二维 0-1（理解用，面试可口述后写一维）**

```python
def knapsack_01_2d(weights, values, cap):
    n = len(weights)
    dp = [[0] * (cap + 1) for _ in range(n + 1)]
    for i in range(1, n + 1):
        w, v = weights[i - 1], values[i - 1]
        for x in range(cap + 1):
            dp[i][x] = dp[i - 1][x]
            if x >= w:
                dp[i][x] = max(dp[i][x], dp[i - 1][x - w] + v)
    return dp[n][cap]
```

C++ 侧函数名 `knapsack01` / `knapsack_unbounded`，循环边界与 Python 相同。

### 变体与技巧

**多重背包**

第 `i` 件最多 `c[i]` 件。朴素拆成 `c[i]` 个 0-1 物品；优化：**二进制拆分**为 `1,2,4,...` 件的组合，物品数变为 `O(log c[i])`，再跑 0-1。总复杂度约 `O(n log C · cap)`。

**分组背包**

物品分成若干组，**每组至多选一件**。在每组内对每件物品做 0-1 更新，组与组之间顺序无关；实现上对每组单独一轮逆序容量循环。

**恰好装满 / 方案数**

- **恰好和是否存在**（494）：`dp[0]=1`，对每个数 `num` 逆序更新 `dp[s] += dp[s-num]`，答案 `dp[target]`。
- **零钱兑换 II**（518）：完全背包求方案数，`dp[0]=1`，正序容量，`dp[x] += dp[x-coin]`。
- **零钱兑换**（322）：完全背包求最少硬币数，`dp` 初始化为 `inf`，正序 `dp[x] = min(dp[x], dp[x-c]+1)`。
- **分割等和子集**（416）：判断是否存在子集和为 `sum/2`，0-1 可达性或 `dp[j]` 布尔滚动。

**依赖背包**

物品有依赖关系（选 B 必须先选 A），需拓扑排序后按依赖顺序分组 DP，竞赛向；面试少见。

**价值与重量对调**

当 `cap` 极大、`v[i]` 很小，可定义 `dp[i][val]` = 达到价值 `val` 的最小重量，答案找最小 `val` 使重量 ≤ `cap`。适用于「价值总和」比容量小得多的题目。转移时在价值维上 0-1 逆序，语义与容量维相同，只是「消耗」从重量变成价值。面试若 `n·cap` TLE 而 `n·sum(values)` 可接受，应主动提出对调。

**分组背包（模板）**

设组 `g` 内有物品集合，每组至多选一件。伪代码：

```text
dp[0..cap] = 0
for each group G:
    tmp = dp 拷贝 或 逆序更新隔离组内贡献
    for each item (w,v) in G:
        for x from cap down to w:
            dp[x] = max(dp[x], tmp[x-w] + v)  # 具体写法多种，核心是组间 0-1、组内选一
```

实现细节因写法而异，关键是**不要**在组内对同一物品循环多次造成「组内选多件」。可先对每组跑一遍 0-1，再进入下一组。

**单调队列优化**

多重背包或某些完全背包在约束下可优化单维转移，OI 向；本页不展开实现，知道存在即可。

**与 `algo-greedy` 的衔接**

- **分数背包**：贪心按密度，见 `algo-greedy`。
- **0-1 贪心失败反例**：容量 10，物品 A(10,10) 与 B(5,6)+C(5,6)，密度 A=1，B=C=1.2，贪心先选 A 得 10，最优选 B+C 得 12。
- **活动选择、区间贪心** 与背包无关，勿混用。

| 416 | 0-1 可达 | 和为 sum/2 |
| 494 | 0-1 计数 | 目标和 ±num |
| 322 | 完全 min | 最少硬币 |
| 518 | 完全 count | 方案数 |
| 474 | 0-1 | 一和 m、一和 n 的双约束（二维容量） |
| 879 | 0-1 | 盈利计划，二维费用 |

**416 分割等和子集（详解）**

给定非空数组 `nums`，能否划分为两个子集，使两子集元素和相等。令 `S = sum(nums)`，若 `S` 为奇数直接 false；否则目标 `target = S/2`。问题变为：是否存在子集和恰好为 `target`（0-1 每件用一次）。`dp[j]` 布尔或「可达」：`dp[0]=True`，对每个 `num` 逆序 `j` 从 `target` 到 `num`：`dp[j] |= dp[j-num]`。答案 `dp[target]`。与 `knapsack_01` 同向循环，但语义是 OR 而非 `max` 价值。

**322 零钱兑换（详解）**

硬币面额 `coins`，总额 `amount`，每种硬币无限枚，求最少硬币数凑出 `amount`，凑不出返回 -1。完全背包求 min：`dp[0]=0`，其余 `dp[x]=inf`，正序 `x`，对每个 `c`：`dp[x]=min(dp[x], dp[x-c]+1)`。注意是 **min** 不是 max；外层可先硬币后容量或先容量后硬币（均正序）。返回 `dp[amount]` 若仍为 `inf` 则 -1。

**494 目标和（详解）**

给 `nums` 与 `target`，每个数前加 `+` 或 `-`，能否使代数和为 `target`。设正子集和 `P`、负子集和 `N`（存正数），则 `P-N=target` 且 `P+N=sum`，得 `P=(sum+target)/2`。转化为选若干数凑成 `P` 的方案数，0-1 计数 DP：`dp[0]=1`，逆序 `dp[j]+=dp[j-num]`。需先判 `sum+target` 为偶且非负等边界。

**分数背包再强调（链向 algo-greedy）**

经典反例：容量 7，物品 (4,9)、(3,5)、(3,5)。密度 2.25、1.67、1.67，贪心先装 4 重量得 9，剩余 3 装一件 3 得 14；最优两件 3 加一件 4？实际最优常为两件 3（重 6 值 10）加部分？——标准反例是容量 50，物品1(10,60) 密度6，物品2(20,100) 密度5，物品3(30,120) 密度4：贪心全装物品1 得 60，最优物品2+3 各取一部分得 240。该题**必须**用 `algo-greedy` 分数背包，本页 `knapsack_01` 会把每件当作不可分割而算错。面试一句话划界：题目没说可分割，默认 0-1。

**多重背包二进制拆分（思路）**

物品 `i` 有 `c[i]` 件，拆成重量价值相同的 1,2,4,..., 余数 件，共 `O(log c[i])` 个伪物品，再跑 0-1。例如 7 件拆成 1+2+4 组合可表示 0..7 任意件数。总复杂度 `O(cap · sum log c[i])`，竞赛常用，面试提到即可。

### 易错点

1. **0-1 内层正序**：同一物品被用多次，答案偏大；完全背包误写逆序则每件最多用一次。
2. **`dp` 语义混淆**：「至多 `cap`」与「恰好 `cap`」初始化不同；恰好不可达应用 `-inf` 或 False，不能一律 `0` 当最大价值。
3. **下标与容量**：`range(cap, w-1, -1)` 在 `w=0` 时需题面保证无零重量物品；零重量物品需单独讨论无限价值或跳过。
4. **答案取错**：有的题要 `max(dp)` 任意容量，有的要 `dp[cap]` 恰好装满。
5. **零钱 322 初始化**：`dp` 除 `dp[0]=0` 外为 `inf`，`min` 转移；与「最大价值」模板不同。
6. **方案数取模**：518 等需对 `10^9+7` 取模，加法转移 `dp[x] += dp[x-w]`。
7. **C++ 类型**：累加和价值用 `long long`，避免中间溢出。
8. **空输入**：Study `knapsack_01([], [], 0)==0`；`cap=0` 应返回 0。
9. **分数背包误用 DP**：可分割题用 DP 浪费且可能错（若强行 0-1 建模）。
10. **多重背包未拆分**：把 `c[i]=100` 直接嵌三层循环 TLE，应二进制拆分。

## Python 实现

Study 文件 `knapsack_dp.py` 实现 0-1 与完全背包及自测。完整源码如下（与仓库一致）：

```python
"""0-1 背包与完全背包。"""

from __future__ import annotations


def knapsack_01(weights: list[int], values: list[int], cap: int) -> int:
    dp = [0] * (cap + 1)
    for w, v in zip(weights, values):
        for x in range(cap, w - 1, -1):
            dp[x] = max(dp[x], dp[x - w] + v)
    return dp[cap]


def knapsack_unbounded(weights: list[int], values: list[int], cap: int) -> int:
    dp = [0] * (cap + 1)
    for x in range(1, cap + 1):
        for w, v in zip(weights, values):
            if x >= w:
                dp[x] = max(dp[x], dp[x - w] + v)
    return dp[cap]


if __name__ == "__main__":
    w = [1, 2, 3]
    val = [6, 10, 12]
    assert knapsack_01(w, val, 5) == 22
    assert knapsack_unbounded([1, 3, 4], [15, 50, 60], 8) == 130
    assert knapsack_01([], [], 0) == 0
    print("knapsack_dp OK")
```

**实现要点**

- `knapsack_01`：外层 `zip(weights, values)` 等价按序处理每件物品；内层 `x` 从 `cap` 到 `w` 递减，保证 0-1 性质。
- `knapsack_unbounded`：外层容量升序，内层尝试所有物品，允许同一容量状态被同一物品多次改进。
- 空列表与 `cap=0`：循环不执行或仅 `dp[0]=0`，返回 0。

运行自测见 **Study 仓库对照** 一节 PowerShell，应打印 `knapsack_dp OK`。

**调试建议**：在 0-1 内层循环打印 `(w,v,x,dp[x])` 观察是否只更新一次；完全背包对固定 `x` 打印多轮 `dp[x]` 应可能连续上升。

**扩展练习（非仓库代码）**：自行实现 322 的 `min` 版完全背包，与 `knapsack_unbounded` 对比初始值与转移函数（`max` vs `min`）。

**逐行阅读 `knapsack_01`**

- `dp = [0] * (cap + 1)`：下标 `0..cap` 均合法；`dp[0]` 表示容量 0 价值 0。
- 外层 `zip`：物品顺序不影响 0-1 最优值（可交换），但影响方案回溯路径。
- `range(cap, w - 1, -1)`：当 `w > cap` 时内层为空，该物品跳过，正确。
- `dp[x] = max(dp[x], dp[x-w]+v)`：不选时 `dp[x]` 不变；选时用「剩余容量 `x-w` 的旧最优」加 `v`。

**逐行阅读 `knapsack_unbounded`**

- 外层 `x` 从 1 开始：容量 0 保持 0；有的题从 0 开始枚举亦可，但需统一语义。
- 内层 `if x >= w`：避免负下标；等价于二维完全背包中「至少用一件」的转移。
- 同一 `x` 可能被多件物品依次改进，正序允许「同一物品贡献多次」的叠加效果。

**与 518 的代码差异**：518 用 `dp[x] += dp[x-coin]`，模 `10^9+7`；本函数用 `max` 求价值。结构同为正序容量，转移运算符不同。

## C++ 实现

C++ 镜像 `knapsack_dp.cpp`，逻辑与 Python 一一对应：

```cpp
#include <alg_std.hpp>
#include <cassert>
using namespace std;

int knapsack01(const vector<int>& w, const vector<int>& v, int cap) {
    vector<int> dp(cap + 1, 0);
    for (int i = 0; i < (int)w.size(); ++i)
        for (int x = cap; x >= w[i]; --x) dp[x] = max(dp[x], dp[x - w[i]] + v[i]);
    return dp[cap];
}

int knapsack_unbounded(const vector<int>& w, const vector<int>& v, int cap) {
    vector<int> dp(cap + 1, 0);
    for (int x = 1; x <= cap; ++x)
        for (int i = 0; i < (int)w.size(); ++i)
            if (x >= w[i]) dp[x] = max(dp[x], dp[x - w[i]] + v[i]);
    return dp[cap];
}

int main() {
    vector<int> w{1, 2, 3}, val{6, 10, 12};
    assert(knapsack01(w, val, 5) == 22);
    assert(knapsack_unbounded({1, 3, 4}, {15, 50, 60}, 8) == 130);
    cout << "knapsack_dp OK" << endl;
    return 0;
}
```

**与 Python 的差异**

- 函数命名 `knapsack01` 无下划线，符合 C++ 风格。
- `vector<int> dp(cap + 1, 0)` 默认零初始化；大 `cap` 注意内存 `(cap+1)*4` 字节量级。
- C++ `main` 未测空向量；`knapsack01({}, {}, 0)` 应返回 0，与 Python 一致。

编译运行见 **Study 仓库对照**；输出 `knapsack_dp OK` 即通过断言。

**对拍**：随机生成小 `n, cap`，暴力 0-1 `O(2^n)` 与 DP 比较；完全背包可用「每种物品无限」的 BFS 或递推对照小数据。

**C++ 实现细节**

- `knapsack01` 使用 `const vector<int>&` 避免拷贝大数组；竞赛可改传参方式。
- 内层 `--x` 与 Python `range(cap, w-1, -1)` 等价；注意 `w[i]==0` 时死循环风险，题面通常保证正重量。
- `knapsack_unbounded` 双重循环与 Python 一致；`max` 两侧类型为 `int`，极大价值可改 `long long`。
- 未包含 `knapsack_01` 空数组测试，建议本地 `assert(knapsack01({}, {}, 0)==0)` 与 Python 对齐。

**双语言对拍脚本思路**：用 PowerShell 调用 Python 生成随机 `w,v,cap`，写临时 JSON，C++ 读入或硬编码同一组数据，比较输出。种子固定便于复现 WA。

「这是 0-1 背包，每件最多一次。定义 `dp[x]` 为容量 x 的最大价值，枚举物品时对容量逆序更新，避免一件用多次。复杂度 O(nW)。若物品可无限用，容量正序。若可分割，不用 DP，用贪心价值密度，见贪心专题。」

### 0-1 逆序的正确性（归纳）

处理物品 `i` 时，更新后的 `dp[x]` 应只考虑物品 `1..i`。更新使用 `dp[x-w]`，若内层正序，则 `dp[x-w]` 可能已在本轮放入物品 `i`，等价物品 `i` 用多次。逆序时，所有 `x' > x` 已更新，而 `x-w < x` 尚未用当前物品更新，故 `dp[x-w]` 仍表示「前 i-1 件」的最优。完全背包正序则允许 `dp[x-w]` 已含当前物品，即重复选取。

### 与打家劫舍的类比

打家劫舍：`dp[i]=max(dp[i-1], dp[i-2]+nums[i])`，相邻不能同抢。0-1 背包：`dp[x]=max(dp[x], dp[x-w]+v)`，「冲突」是容量而非下标。两者都是一维滚动，但背包的「维度」是资源消耗， robbery 的维度是时间下标。

### 常见 Follow-up

- **输出具体选了哪些物品？** 另开 `choice[x]` 或从 `dp` 回溯：若 `dp[x]==dp[x-w]+v` 则选了该件。
- **价值最大且重量最小？** 双关键字 DP 或第二维记录重量。
- **多背包？** 多维容量或枚举分配。
- **cap 超大？** 换价值 DP 或 meet-in-the-middle，见变体节。

### 识别速查（考前一页）

- 每件最多一次、求最大价值 → 0-1 逆序。
- 每件无限、求最大价值或最少件数 → 完全正序（min 换 inf 初始化）。
- 子集和、划分 → 0-1 可达或计数。
- 物品可分割 → `algo-greedy` 分数背包，非本页。

### 416 手推（nums=[1,5,11,5]）

`sum=22`，`target=11`。`dp[0]=True`。处理 1：`dp[1]=True`。处理 5：`dp[6],dp[5]` 等可达。处理 11：可达 11。处理 5：不增加新可达。最终 `dp[11]=True`，可划分。若 `nums=[1,2,3,5]`，`sum=11` 奇数，直接 false。

### 322 手推（coins=[1,2,5], amount=11）

`dp[0]=0`，其余 inf。正序凑出：`dp[1]=1,d[2]=1,d[3]=2,d[4]=2,d[5]=1,...` 最终 `dp[11]=3`（5+5+1）。与完全背包 min 转移一致。

### 二维费用背包（474 思路）

`dp[i][j]` 表示用了 `i` 个 0、`j` 个 1 的最大字符串个数（或能否达到）。对每个字符串 `(a,b)` 代价 `(a,b)`，0-1 更新：逆序 `i,j` 双重循环，`dp[i][j]=max(dp[i][j], dp[i-a][j-b]+1)`。是 0-1 背包在二维容量上的直接推广，Study 本页未实现，但循环方向仍为「费用从大到小」。

### 排列型完全背包（377 区别）

377 求硬币组合**排列数**（顺序不同算不同），外层枚举容量、内层枚举硬币的「完全背包」统计的是**组合**（顺序无关）。排列型需外层枚举物品、内层正序容量，或定义 `dp[x]` 时先遍历硬币再更新。刷 518 后再做 377，避免混模板。

### 正确性一句话

0-1 逆序：归纳证明处理物品 `i` 后 `dp[x]` 仅含前 `i` 件。完全正序：允许 `dp[x-w]` 已含物品 `i` 多次，归纳证明 `dp[x]` 为允许无限件的最优。分数背包贪心：交换论证，与 DP 无关。

### 0-1 与完全对照实验

建议本地打印处理单件物品 `(w=2,v=10)`、`cap=5` 时两种顺序的 `dp` 数组：

- **逆序 0-1**：处理前 `[0,0,0,0,0]`，处理后 `[0,10,10,10,10]`——每个容量至多吸收一次该物品。
- **错误正序 0-1**：可能得到 `[0,10,20,20,20]`，`dp[4]` 被同一物品更新两次，等价完全。
- **正序完全**：允许 `dp[4]=20`，符合无限件语义。

亲眼对比一次，胜过背诵十遍口诀。完全背包 Study 样例 `cap=8` 建议在 `knapsack_unbounded` 末尾临时打印 `dp` 数组，对照 130 的组成。

### 回溯输出所选物品（0-1）

在 `knapsack_01` 外维护 `choice[x]`：当 `dp[x]` 由 `dp[x-w]+v` 改进时记录物品 id。从 `x=cap` 开始，若 `choice[x]!=-1`，则选中该物品并令 `x -= w[id]`，否则 `x` 不变或表示未使用（依实现而定）。回溯复杂度 `O(cap)`。面试 Follow-up 常考，需在二维版理解后再压到一维。

### 幂次与二进制拆分补充说明

多重背包将件数 `c` 拆为 `1,2,4,...,2^k` 与余数，是因为二进制表示可覆盖 `0..c` 任意选取件数；拆完每件变成 0-1 物品后，总物品数约 `O(n log C)`，再对容量逆序。竞赛若出现「每种最多三件」，也可直接展开三件 0-1，面试小数据可行。理解拆分有助于解释「为何多重最终仍用 0-1 逆序」——本质没有新的循环方向，只是物品列表变长。

### 最后提醒

写背包题时，**先写方向再写循环体**；许多 WA 不是转移式错，而是 `for x in range(cap, w-1, -1)` 写成了 `range(w, cap+1)`。把这两行在模板中用注释标出「0-1 逆序 / 完全正序」，可显著降低失误率。与 `algo-greedy` 对照时，重点记住：可分割 → 排序贪心；不可分割 → 本页 DP。两条规则覆盖绝大多数背包类题面表述。
