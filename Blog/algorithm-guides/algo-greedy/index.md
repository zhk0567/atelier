# 算法 · 贪心（Greedy）

## 导读

**贪心**在每一步做**当前看起来最优**的局部决策，不回头、不枚举全局组合，期望局部最优的拼接就是全局最优。与动态规划的区别：DP 要比较「选 / 不选」等多种子结构并填表；贪心在证明成立时**只保留一条决策链**，通常排序一次再线性扫描。与分治的区别：分治合并子问题解；贪心往往**无显式合并**，只有排序键与扫描顺序。

Study 仓库 `greedy/` 提供两个可运行模板，覆盖贪心证明里最常见的两类骨架：

| 函数 | 含义 | 典型题 |
|------|------|--------|
| `activity_selection` | 最多不相交区间数 | 435、452、会议室 II |
| `fractional_knapsack` | 可分割背包最大价值 | 经典教材题、与 0-1 对比 |

本页在 `notes.md`「活动选择 + 分数背包」基础上，系统讲解**交换论证**、**拟阵（matroid）直觉**、**区间调度族**（按结束时间 / 按开始时间）、**Huffman 合并模式**、**硬币凑数反例**，以及面试中「先问能否分割、再选贪心还是 DP」的答题脚本。`greedy.py` / `greedy.cpp` 带 `__main__` 断言，适合 PowerShell `-LiteralPath` 本地对拍后再刷 LeetCode。

## 基础篇

### 直觉与定义

**贪心算法**不定义统一「状态表」，而是：**排序** → **按序扫描** → **若当前选择合法则接受**。正确性依赖问题结构，不能凭直觉声称「局部最优 = 全局最优」。工程上常用两条证明路线：

**交换论证（exchange argument）**  
设贪心序列为 `G`，某最优解为 `O`。若 `G` 与 `O` 首个不同选择出现在位置 `k`，证明可以把 `O` 在 `k` 处的选择**换成**贪心选择且不劣化（通常不变或更优），重复即可将 `O` 转化为 `G`。活动选择按**结束时间**排序：若最优解第一个区间不是结束最早的可选区间，用结束更早的区间替换不会减少后续可选数量（因为结束更早释放资源更早）。分数背包按**密度降序**：若最优解先装了低密度物品，把等量重量从低密度换到高密度，价值不会下降。

**拟阵直觉（matroid intuition）**  
拟阵 `(E, I)` 中 `I` 为独立集族，满足：空集独立；子集封闭；**交换性**——若 `A,B` 独立且 `|A|<|B|`，则存在 `e∈B\A` 使 `A∪{e}` 独立。在**带权拟阵**上，按权重从高到低贪心选入（保持独立）得到最大权独立集。活动选择的独立集是「两两不重叠的区间子集」，按结束时间贪心等价于按「结束时间作权重」的特殊情形。最小生成树 Kruskal：边集独立当且仅当无环，按边权升序贪心选边即拟阵贪心。**不必考拟阵形式化证明**，面试可说：「问题有 matroid 结构时，排序贪心正确。」

**活动选择（Activity Selection）**  
输入区间 `(start, end)`，选最多不相交区间。贪心：按 `end` 升序，能选则选并更新 `last_end`。Study `activity_selection` 用 `s >= last_end` 允许贴边相接。手算 `[(1,2),(3,4),(5,6),(7,8)]`：四个互不重叠，全选得 4。若 `[(1,4),(2,3),(4,5)]`，按结束排序 `(2,3),(1,4),(4,5)`，选 `(2,3)` 后 `last=3`，`(1,4)` 因 `1<3` 跳过，`(4,5)` 可选，共 2 个。

**分数背包（Fractional Knapsack）**  
物品可分割，按 `v/w` 从高到低装入直至容量满。Study 断言：重量 `(10,20,30)`、价值 `(60,100,120)`、容量 50 → 先装 10 得 60，再装 20 得 100，再装 20/30 的 120 的一部分，总计 240。交换论证：任意最优解中若存在「已装低密度、未装满高密度」，交换重量后价值上升。

**0-1 背包不能用分数贪心**  
物品 `(1,10),(2,20),(3,30)` 容量 5：密度均为 10，贪心按密度与 DP 可能同为 50；反例在**不可分割**：`(4,9),(3,5),(3,5)` 容量 7，贪心先装 4 得 9，剩余装一件 3 得 14；最优两件 3 重 6 值 10 再加部分 4？标准教学反例是容量 50 三物品：贪心只装第一件 60，最优 2+3 各取一部分 240——见下文 **Python 实现** 断言。划界：**题面未写可分割，默认 0-1，转 `algo-dp-knapsack`**。

**硬币凑数反例**  
面值 `[1,3,4]` 凑 6：贪心每次取最大面额 → `4+1+1` 共 3 枚；最优 `3+3` 共 2 枚。故 **322 最少硬币** 用完全背包 `min` DP，不能对任意面值贪心。若面值满足「规范币制」（如人民币 1,2,5,10…）可贪心，但面试默认任意面值。

**Huffman 编码（模式，非 Study 文件）**  
合并代价最小的二叉树：每次合并频率最小的两个结点，等价于「全局最优由局部最小合并」的贪心。与活动选择「按结束释放资源」不同，Huffman 是**配对合并** + 优先队列。LeetCode 1167 连接棒、973 最近原点、295 中位数堆变体同属「反复取最小两个合并」。

**区间调度变体**  
- 最多活动数：按**结束时间**（本仓库）。  
- 最少移除覆盖：常转化为最多保留，或按结束排序贪心。  
- 最多重叠层数 / 会议室 II：按**开始时间**排序，小根堆维护结束时间（贪心 + 堆）。  
- 435 无重叠区间：按结束排序，保留则跳过冲突，等价活动选择。  
- 452 用最少的箭：按结束排序，箭穿过区间右端点贪心。

**任务调度**  
单机器、每个任务 `(start, end, profit)` 选子集最大化利润且不相交：按结束排序 + 二分找上一个不冲突 + DP 或贪心+结构（部分题需 DP）。纯「单位时间权重」且可并行则不同模型。

### 复杂度分析

| 模板 | 时间 | 空间 | 说明 |
|------|------|------|------|
| 活动选择 | O(n log n) | O(1) 或 O(n) 排序 | 排序主导 |
| 分数背包 | O(n log n) | O(1) | 忽略 w≤0 |
| Huffman | O(n log n) | O(n) 堆 | n 次合并 |
| 会议室 II | O(n log n) | O(n) 堆 | 按开始时间 |
| Kruskal MST | O(E log E) | 并查集 | 拟阵贪心 |

面试应说明：贪心常**不**比 DP 渐近更优，而是**证明成立时实现更短**；硬币、0-1 背包贪心可能错，复杂度再低也无意义。

**摊还**：扫描指针 `last_end` 只增，每个区间 O(1) 判断；堆维护会议室层数，每个区间最多入堆出堆各一次。

### 代码模板

**活动选择（与 Study 一致）**

```python
def activity_selection(intervals):
    intervals = sorted(intervals, key=lambda x: x[1])
    cnt, last_end = 0, -10**9
    for s, e in intervals:
        if s >= last_end:
            cnt += 1
            last_end = e
    return cnt
```

**分数背包**

```python
def fractional_knapsack(items, capacity):
    ratio = [(v / w, w, v) for w, v in items if w > 0]
    ratio.sort(reverse=True)
    rem, total = float(capacity), 0.0
    for _r, w, v in ratio:
        if rem <= 0:
            break
        take = min(rem, w)
        total += take * (v / w)
        rem -= take
    return total
```

**会议室 II（堆，仓库外模板）**

```python
import heapq
def min_meeting_rooms(intervals):
    intervals.sort(key=lambda x: x[0])
    heap = []
    for s, e in intervals:
        if heap and heap[0] <= s:
            heapq.heappop(heap)
        heapq.heappush(heap, e)
    return len(heap)
```

**Huffman 合并骨架**

```python
import heapq
def huffman_cost(freq):
    h = list(freq)
    heapq.heapify(h)
    cost = 0
    while len(h) > 1:
        a, b = heapq.heappop(h), heapq.heappop(h)
        c = a + b
        cost += c
        heapq.heappush(h, c)
    return cost
```

**硬币贪心（仅当确认规范币制时）**

```python
def coin_greedy(coins, amount):
    coins = sorted(coins, reverse=True)
    cnt = 0
    for c in coins:
        if amount <= 0:
            break
        cnt += amount // c
        amount %= c
    return cnt if amount == 0 else -1
```

对 `[1,3,4]` 凑 6 返回 3，非最优；322 用 DP。

**Huffman 与合并类**  
1167 连接棒、1000 合并石头（石子合并是区间 DP，单链合并可用 Huffman 思想区分）、295 数据流中位数。识别词：**每次合并两个最小/最大代价**。

**区间 DP vs 贪心**

### 易错点

1. **0-1 当分数**：对不可分割物品按密度装，WA。  
2. **硬币任意面值贪心**：322 必须用 DP；`[1,3,4]` 凑 6 是经典反例。  
3. **活动选择按开始时间排序**：一般错；应按**结束时间**（或题设等价变换）。  
4. **重叠判定**：`s < last_end` 与 `s >= last_end` 与题意「严格/非严格」不一致。  
5. **weight≤0**：Study 分数背包跳过 `w<=0`；负权需另建模。  
6. **空输入**：`activity_selection([])==0`，`fractional_knapsack([], cap)==0`。  
7. **capacity=0**：分数背包返回 0。  
8. **会议室 II 忘排序开始时间**：堆逻辑错乱。  
9. **Huffman 只合并一次**：需循环至剩一个结点。  
10. **证明缺失**：面试只写排序不写「为何按 end」会失分。  
11. **435 与 452 混淆**：一个删区间一个射箭，排序键类似但输出不同。  
12. **可分割未读题**：LeetCode 502 等需读清题意。  
13. **浮点精度**：分数背包比较用 `v/w` 或交叉乘 `v1*w2` vs `v2*w1` 避免除法误差。  
14. **多机调度**：同一机器才用活动选择；并行机是另一问题。  
15. **把 MST 当活动选择**：Kruskal 是边权排序 + 并查集，独立集是森林。

### 交换论证详解：活动选择

设区间按结束时间非降排列。贪心依次选第一个与已选不冲突的区间。设最优解 `OPT` 与贪心 `G` 第一个不同在区间 `i`：`G` 选了结束最早的可用 `a`，`OPT` 选了 `b` 且 `end(a) <= end(b)`（因排序）。在 `OPT` 中用 `a` 替换 `b`：因 `a` 结束不晚于 `b`，`OPT` 中与 `b` 冲突的区间与 `a` 冲突集合只可能更小，替换后仍可行且数量不变。归纳得存在与 `G` 同样大小的最优解，故贪心最优。

### 交换论证详解：分数背包

设最优解按装入顺序排列。若存在相邻两段密度 `d1 < d2` 且两段都部分或全部装入，交换相同总重量使高密度多装、低密度少装，总价值不减。反复交换后得到按密度非增顺序，与贪心一致。故贪心达到最优值。

### 拟阵与 Kruskal（延伸）

图 `G=(V,E)`，独立集为无环边集。Kruskal 按权升序加边，若不成环则加入。拟阵交换性保证：若当前森林 `F` 与某最优树 `T` 边数相同且不同，可逐步用更轻且不环的边替换。这是「排序 + 扫描」的另一实例，与区间活动选择共享**证明模板**，面试可一句话带过 MST。

### 硬币与规范币制

**任意面值**：贪心失败，`[1,3,4]` 凑 6。  
**规范币制**（每个面额大于前面面额之和等条件）：贪心可行，但笔试很少直接给这种条件。  
**322**：`dp[x] = min(dp[x-c]+1)`，完全背包最少枚数，初始化 `inf`。  
**518**：完全背包方案数。  
面试话术：「若未证明币制，我用 DP；若题目说明贪心硬币可行，再写从大到小取。」

### Huffman 正确性直觉

每次合并两个最小频率结点，合并代价累加；局部「先合并最小」使后续大频率结点深度增加的总代价最小。交换论证：若最优合并树中存在某内部结点合并的不是当前最小两个，可交换子树降低该子树贡献。与活动选择「尽早释放资源（早结束）」对偶：**Huffman 尽早合并小权重**。

### 活动选择课堂笔记（扩充）

活动选择问题是贪心教学的开场题，因为它**不**需要 DP 表，却需要**严格证明**。许多初学者按「开始时间」排序，先选开始最早的区间，反例立刻出现：区间 `[(0,10),(1,2),(2,3)]` 若先选 `(0,10)` 则只能再选 0 个，而按结束时间选 `(1,2),(2,3)` 可得 2 个。记住口诀：**「尽早结束」等价于「尽早释放资源」**。在会议室、CPU 时间片、教室排课等模型里，资源就是时间轴，结束越早，后面可用窗口越长。

形式化地，设区间闭区间 `[s_i, e_i]`，若 `s_j >= e_i` 则称 i 与 j 不冲突（Study 实现）。部分题目要求严格不等 `s_j > e_i`，只需改比较符，排序键仍通常是 end。435 题输出「移除区间的最小数量」，等于 `n - activity_selection(保留)`，保留策略与活动选择相同：排序后贪心保留不冲突区间。

452 引爆气球可视为「箭穿过区间右端点」：按 end 排序，当前箭位置取当前区间 end，凡 start ≤ 箭位置的区间被同一箭覆盖，跳过直到 start 更大。与活动选择共享排序键，但输出是箭数而非保留数。面试时先画时间轴，标出 end 排序后的扫描过程，考官更容易跟上。

加权活动选择（带利润）不能纯贪心：需按 end 排序后 `dp[i] = max(dp[i-1], profit[i] + dp[p(i)])`，`p(i)` 为最后一个与 i 不冲突的下标（二分）。这是贪心排序 + DP 的混合，说明「排序键来自贪心，全局最优仍需子问题合并」。本页 Study 只做无权计数，加权留到 DP 专题。

### 分数背包课堂笔记（扩充）

分数背包是「线性规划松弛」在离散竞赛里的最简单形态：变量 `x_i ∈ [0,1]` 表示装入比例，目标 `max Σ x_i v_i`，约束 `Σ x_i w_i ≤ W`。最优解一定在顶点（每个 `x_i` 为 0 或 1 或「卡容量」），故按密度贪心即可。若所有 `w_i` 相同，退化为按价值排序；若所有 `v_i` 相同，按重量小的优先（密度大）。

浮点实现中，比较 `v1/w1` 与 `v2/w2` 可用交叉乘法 `v1*w2` 与 `v2*w1` 避免除法误差。竞赛大数据可用 `Decimal` 或分数类，Study 用 `float` 足够通过 `1e-9` 断言。容量极大时注意 `rem` 类型与溢出，C++ 用 `double` 一般安全，若价值和需 `long double` 读题。

与 502 IPO 等题的区别：502 常结合堆与贪心选项目，不完全是分数背包模板，但「按某种比率排序后依次取」的精神相近。读题时画出「能否只取一部分」——题干出现「比例」「分割」「partial」等词即高度怀疑分数背包。

### Huffman 编码课堂笔记（扩充）

Huffman 树构造：维护最小堆，反复取出两个最小频率结点合并，新结点频率为二者之和，代价累加（LeetCode 1167 即此模型）。正确性：高频字符应处于较浅深度，低频应较深；若某棵子树中交换深浅，总代价 `Σ freq * depth` 不会更优。与活动选择「选 end 最小」对偶：Huffman「合并 freq 最小」。

编码表生成：从堆中记录左右子，DFS 得 0/1 码字。平均码长 `Σ p_i * len(code_i)` 最小。扩展：算术编码、ANS 不在本页，但识别「合并两最小」题型的关键词一致。

973 最近 K 点：按坐标轴分治或堆，不完全是 Huffman，但「选极值合并」可练堆熟练度。295 中位数：双堆维护左右半，是「平衡极值」而非 Huffman，勿混模板。

### 拟阵与贪心族谱（扩充）

除活动选择、Kruskal 外，还有：**任务调度**（单位时间、截止期，EDF 在单机有时最优）、**最小延迟调度**（按 Smith 规则 `w/p` 排序）。这些都有专门条件，不能生搬。拟阵给你的是「遇到独立集 + 权重，先想排序贪心」的检查清单。

**非拟阵例子**：0-1 背包的独立集（子集重量和 ≤ W）一般**不是**拟阵，故密度贪心失败。图的最短路径带负权也不能 Dijkstra 贪心。记住：**拟阵是充分条件，不是必要条件**；有些非拟阵问题仍有贪心，需单独证明（如部分区间题）。

### 交换论证书写模板（面试白板）

第一步：定义贪心顺序 `g_1, g_2, …`。第二步：设最优 `o_1, o_2, …` 按同一排序键排列（若最优解可重排为相同排序而不减目标）。第三步：找第一个 `k` 使 `g_k ≠ o_k`。第四步：构造 `o'` 在 `k` 用 `g_k` 替换 `o_k`，证明 `|o'|` 或价值 ≥ `|o|`。第五步：归纳。活动选择在第四步用「end 更早不减少后续空间」；分数背包用「密度更高不减少价值」。

### 硬币问题深度（扩充）

**反例族**：`[1,3,4], 6`；`[1,5,10,25]` 对某些 amount 在特定国家币制外仍可能失败，面试以 `[1,3,4]` 为准。**322 状态**：`dp[x]` 为凑 x 的最少枚数，`dp[0]=0`，其余 `inf`，完全背包正序 `min`。与贪心 `while amount: take largest coin` 对比写三行代码即可展示差异。

### 区间调度综合讲义（扩充）

**986 区间交集**：双指针，非纯贪心排序单键，但需按 start 排序。**1288 删除覆盖**：按 end 排序，类似活动选择保留。**57 插入区间**：插入后 56 合并。**763 字母区间**：维护 `[left,right]` 扩展，贪心扩 right 至最后出现位置。

### 跳跃游戏讲义（扩充）

**55**：维护 `far` 最远可达，若 `i > far` 失败，否则 `far = max(far, i+nums[i])`。O(n) 贪心，正确性：若 i 可达，则 `[0,i]` 内某点能跳到 i，只需关心最远边界。**45**：在 55 基础上，当 `i` 到达当前步边界时步数加一，边界设为 `far`。与 BFS 层数等价。**134 加油站**：总油量 ≥ 总消耗则解存在；从唯一可能起点（累计和最小点）模拟，贪心选起点。

### Dijkstra 与贪心（扩充）

非负权图单源最短路：每次取出未确定结点中 dist 最小者，松弛邻边。这是**贪心选择性质**（正确性依赖非负权）。与活动选择不同，Dijkstra 是「逐步固定最优」而非一次排序扫描。面试若问「贪心图算法」，可答 Dijkstra、Kruskal、Prim（后两者是拟阵/割性质）。

### 双指针与贪心边界（扩充）

**11 盛水**：双指针夹逼，移动较短边，证明移动短边不会漏最优。**15 三数之和**：排序 + 双指针，去重。这些常归类「双指针」而非贪心，但证明也是「若跳过某类移动会漏解」。本页以排序+单指针扫描为主，双指针题在 `algo-two-pointers` 可交叉阅读。

### 交换论证逐步演练（活动选择长稿）

设区间已按结束时间非降排列为 `a_1,…,a_n`。贪心算法 `G` 依次选择第一个满足 `start ≥ last_end` 的区间。设 `O` 为任意最优解对应的区间集合，`|O|` 最大。若 `G` 与 `O` 不完全相同，令 `k` 为最小下标使得前 `k-1` 步贪心选择与 `O` 中对应（按排序后顺序）一致，而第 `k` 步不同。设贪心选 `a`，`O` 在同样位置选 `b`，且 `a` 为当前可用中 end 最小者，故 `end(a) ≤ end(b)`。在 `O` 中用 `a` 替换 `b`：凡与 `b` 冲突的区间，因 `end(a) ≤ end(b)`，与 `a` 冲突者不比与 `b` 冲突者更多；不与 `b` 冲突的区间自然不与更早结束的 `a` 冲突。故新集合仍独立且大小仍为 `|O|`。重复替换，最终得到与 `G` 相同的集合，故 `|G|=|O|`。以上五步可在白板压缩为三十秒口述。

### 交换论证逐步演练（分数背包长稿）

设最优解给定各物品装入比例 `x_i ∈ [0,1]`，总重量 ≤ W。若存在 `i,j` 且 `v_i/w_i < v_j/w_j`，而 `x_i>0` 且未装满 `j`（或 `x_j` 未满），则将微量重量从 `i` 转移到 `j`，总价值增加（密度高者承接重量更划算）。有限次转移后，所有正比例物品按密度非增排列，且满容量前均为密度最高者填满——即贪心解。故贪心价值等于最优值。注意：若所有正比例物品密度相等，任意排列价值相同，贪心仍最优。

### 拟阵形式化一页纸（选读）

拟阵 `M=(E,I)`：`I` 子集族，① ∅∈I；② 若 A∈I 且 B⊆A 则 B∈I（遗传性）；③ 若 A,B∈I 且 |A|<|B|，则 ∃e∈B\A 使 A∪{e}∈I（交换性）。加权拟阵：权重 `w(e)`，贪心按 `w` 降序尝试加入（保持独立）得最大总权独立集。图形拟阵：森林边集独立；Kruskal 即此。区间拟阵：时间轴上区间子集无重叠为独立，活动选择是「卡片inality 最大化」变体，与最大权拟阵表述略有差别但交换精神一致。竞赛不要求默写公理，但提到「拟阵」可展示理论深度。

### Kruskal 与 Prim 对照（图论贪心）

Kruskal：边权升序，并查集判环。Prim：从起点扩展最小边。二者在连通无向图上均给 MST。与活动选择对比：边权排序 vs 区间 end 排序；并查集 vs 单个 `last_end` 指针。面试「最小生成树」问 Prim 还是 Kruskal：稠密图 Prim+堆，稀疏 Kruskal O(E log E)。都不是 Study `greedy.py` 内容，但属贪心证明族谱。

### Dijkstra 正确性草图（图论贪心）

非负权：设 `S` 为已确定最短距离结点集，`u` 为 `V\S` 中 dist 最小者，则 `u` 的最短路已确定。否则存在更短路径经 `V\S` 中某 `x`，可导出矛盾。每次选 dist 最小是贪心选择。负权边需 Bellman-Ford，不能 Dijkstra。最短路+贪心与区间调度不同域，但一起记忆可覆盖「图论贪心」面试点。

### 任务调度 Smith 规则（单机）

处理时间 `p_i`，权重 `w_i`，最小化加权完成时间和。Smith 规则：按 `w_i/p_i` 降序加工（某些模型）。与分数背包密度排序形似但目标函数不同。遇到「调度」题先读是单机还是多机、是否可抢占、优化目标是最小延迟还是最大利润。

## Python 实现

Study 仓库 `greedy.py` 全文如下，含 `__main__` 断言。阅读顺序：先看 `activity_selection` 的排序键与 `last_end`，再看 `fractional_knapsack` 的 `v/w` 排序与 `take = min(rem, w)`。

```python
"""贪心：活动选择、分数背包。"""

from __future__ import annotations


def activity_selection(intervals: list[tuple[int, int]]) -> int:
    """intervals: (start, end)，最多不相交区间数。"""
    intervals = sorted(intervals, key=lambda x: x[1])
    cnt = 0
    last_end = -10**9
    for s, e in intervals:
        if s >= last_end:
            cnt += 1
            last_end = e
    return cnt


def fractional_knapsack(items: list[tuple[int, int]], capacity: int) -> float:
    """items: (weight, value)，可分割，求最大价值。"""
    ratio = [(v / w, w, v) for w, v in items if w > 0]
    ratio.sort(reverse=True)
    rem = float(capacity)
    total = 0.0
    for _r, w, v in ratio:
        if rem <= 0:
            break
        take = min(rem, w)
        total += take * (v / w)
        rem -= take
    return total


if __name__ == "__main__":
    assert activity_selection([(1, 2), (3, 4), (5, 6), (7, 8)]) == 4
    assert activity_selection([]) == 0
    assert abs(fractional_knapsack([(10, 60), (20, 100), (30, 120)], 50) - 240.0) < 1e-9
    assert fractional_knapsack([], 10) == 0.0
    assert fractional_knapsack([(0, 10)], 5) == 0.0
    print("greedy OK")
```

**逐行说明**

- `activity_selection`：`key=lambda x: x[1]` 按结束时间排序；`last_end` 初值极小保证第一个区间可选；`s >= last_end` 表示与上一选中区间不重叠（含贴边）。  
- `fractional_knapsack`：过滤 `w>0`；按 `v/w` 降序；`take` 为剩余容量与物品重量的较小值；累加 `take * (v/w)` 即装入价值。  
- 断言 240.0：与教材标准三物品例一致，用于对拍手写计算。  
- `w=0` 物品被忽略，避免除零；价值 10 的零重物品不贡献。

**本地扩展练习（不改仓库）**  
可在 `__main__` 临时加：`assert activity_selection([(1,4),(2,3),(4,5)]) == 2`；硬币贪心对 `[1,3,4], 6` 得 3 枚，加深反例印象。提交 Study 上游时请保留原断言。

## C++ 实现

`greedy.cpp` 与 Python 逻辑镜像：活动选择按 `pair` 的 `second` 排序；分数背包用 `tuple<double,int,int>` 存密度并 `greater` 排序。

```cpp
#include <alg_std.hpp>
#include <cassert>
using namespace std;

int activity_selection(vector<pair<int, int>> iv) {
    sort(iv.begin(), iv.end(), [](auto& a, auto& b) { return a.second < b.second; });
    int cnt = 0, last = INT_MIN / 2;
    for (auto [s, e] : iv)
        if (s >= last) {
            ++cnt;
            last = e;
        }
    return cnt;
}

double fractional_knapsack(vector<pair<int, int>> items, int cap) {
    vector<tuple<double, int, int>> r;
    for (auto [w, v] : items)
        if (w > 0) r.push_back({(double)v / w, w, v});
    sort(r.begin(), r.end(), greater<tuple<double, int, int>>());
    double rem = cap, tot = 0;
    for (auto [rv, w, v] : r) {
        if (rem <= 0) break;
        double take = min(rem, (double)w);
        tot += take * ((double)v / w);
        rem -= take;
    }
    return tot;
}

int main() {
    assert(activity_selection({{1, 2}, {3, 4}, {5, 6}, {7, 8}}) == 4);
    assert(abs(fractional_knapsack({{10, 60}, {20, 100}, {30, 120}}, 50) - 240.0) < 1e-9);
    cout << "greedy OK" << endl;
    return 0;
}
```

**与 Python 对照**

| 点 | Python | C++ |
|----|--------|-----|
| 排序键 | `x[1]` | `a.second < b.second` |
| 初值 last | `-10**9` | `INT_MIN/2` |
| 密度排序 | `sort(reverse=True)` | `greater<tuple<...>>` |
| 容量类型 | `float` rem | `double rem` |

竞赛提交注意 `long long` 与 `double` 是否与题意一致；面试双语言可只写一种并口述另一种对应行。

### LeetCode 435 与 452 详解（练习延伸）

**435 无重叠区间**：给定区间集合，移除最少区间使剩余互不重叠。等价于保留最多互不重叠区间。算法：按 end 升序排序，贪心保留 `s >= last_end` 的区间并更新 `last_end = e`。答案 `n - 保留数`。与 `activity_selection` 完全一致，仅输出变换。实现时注意：若输入为空返回 0；若只有一个区间返回 0 移除。时间 O(n log n)。

**452 用最少的箭引爆气球**：区间 `[xstart, xend]`，箭可穿过 `[x, x]`（竖直线）。贪心：按 end 排序，第一支箭放在第一个区间 end，之后若 `xstart > 当前箭位置` 则需要新箭并放在新区间 end。与活动选择「选 end 最小释放」同构。手算 `[[10,16],[2,8],[1,6],[7,12]]`：排序后按 end 合并组，常见答案 2 支箭。面试画数轴说明箭位置。

### 253 会议室 II 详解

区间 `[start, end)` 或闭区间依题意。按 **start** 升序，小根堆存当前进行中的会议 **end**。新会议到来：若堆顶 end ≤ start，弹出（房间复用），再 push 新 end；否则 push。堆大小即同时进行会议数最大值。与活动选择区别：**活动选择按 end 排序单次扫描**；**会议室按 start 排序 + 堆**。勿混模板。复杂度 O(n log n)。变体：1094 拼车，站点差分 + 排序事件也可。

### 1167 连接棒与 Huffman 代价

每次合并两根棒，代价为长度之和，求最小总代价。将小根堆，反复 pop 两个最小 push 和，累加代价。与 Huffman 合并频率完全相同。手算 `[2,7,4,5]`：先 2+4=6 代价 6；再 5+6=11 代价 11；再 7+11=18 代价 18；总 6+11+18=35（合并顺序可能略有不同，总代价一致）。代码用 `heapq` 或 `priority_queue`。

### 322 与贪心对比专题（练习延伸）

完整 DP：`dp[0]=0`，`dp[i]=min(dp[i-c]+1)` 对所有 coin，`dp` 初值 inf。贪心 `coin_greedy` 在 `[1,3,4],6` 得 3。对拍程序：随机 amount≤20，枚举 DFS 最优与贪心比，找反例存储。理解「贪心错」比背反例更重要：因为**局部取大面额**无法预见需要用小面额凑整。

扩展：279 完全平方数、139 单词拆分是 DP/搜索，不是贪心。518 方案数是 sum DP。面试连环问：322 最少枚数 → 518 方案数 → 零钱兑换 IV 排列 → 均需 DP 或 BFS，不是硬币贪心。

### 55 与 45 跳跃专题

**55**：`far` 维护最远，遍历 i，若 i>far 返回 false，否则更新 far。正确性归纳：所有可达位置不超过 far。**45**：`end` 为当前步结束位置，到达 end 时 steps++，end 更新为 far。O(n) 时间 O(1) 空间。与 BFS 层序等价。134 加油站：total 若 <0 无解；cur 累加，cur 最小处后一位为起点。

### 406 重建队列专题

排序：`sort(people, key=lambda x: (-x[0], x[1]))`，按身高降序、人数升序。依次插入列表位置 k（人数）。正确性：高个子先占位，矮个子插入不影响已占位高个的相对顺序。贪心在「插入位置」而非区间。面试易忘第二关键字升序。

### 621 任务调度器专题

桶思想：最多出现次数 `M` 的任务至少占 `M-1` 行每行 `n+1` 格加最后一行。答案 `max(len(tasks), (M-1)*(n+1)+count_M)`。构造性，不是排序扫描，但属「公式贪心」。与活动选择模板不同，识别关键词「冷却 n」。

### 763 划分字母区间专题

`last[c]` 记录字符最后出现位置。扫描 i，扩展 `right = max(right, last[s[i]])`，当 `i==right` 时切一段。得到划分段数最少？实际是「尽可能早切」得到合法划分。贪心扩展右边界。

### 56 合并区间专题

按 start 排序，扫合并：若当前区间 start ≤ 上一 end，合并 end；否则 push 新段。与活动选择「选最多」不同，是「合并重叠」。仍属排序后线性扫描贪心族。

### 502 IPO 阅读提示

资本 k，项目 (capital, profit)，选 capital≤k 的项目使 k 最大。常用堆：按 capital 排序，将小根堆存可做项目 profit，pop 最大 profit 更新 k。是「排序 + 堆」而非 Study 两函数，但贪心精神是「每次选当前最优可执行项」。

### 860 柠檬水找零专题

5 元收 10 找 5；20 优先找 10+5 否则 3×5。模拟贪心，维护 5 和 10 库存。证明：20 时若有 10 则必须用 10+5 否则后续 10 找零失败。小额模拟题练手写。

### 134 加油站专题

若总油 < 总耗，无解 -1。否则必存在解。从 0 累加 cur，cur 最小时下一个站点为起点。O(n) 贪心，正确性：若从 A 出发能绕一圈，从 B（A 之后累计和最小点）出发也能。

### 135 分发糖果专题

两次扫描：左到右若评分高则糖果+1；右到左同理；取 max。不是单一排序键，是「双向贪心」套路，与区间调度不同但常考。

### 767 重构字符串专题

按频率排序，隔位插入或用堆每次取最多且与前不同字符。构造贪心，失败返回空串。频率 > (n+1)/2 无解。

### 630 课程表 III（堆）

按 end 排序，堆维护已选课程 duration，若 sum+duration>end 弹出最长 duration。贪心「按截止时间选最多」的堆版本。

### Huffman 实现练习（Python 片段，非仓库）

```python
import heapq
def connect_sticks(sticks):
    h = sticks[:]
    heapq.heapify(h)
    cost = 0
    while len(h) > 1:
        a, b = heapq.heappop(h), heapq.heappop(h)
        s = a + b
        cost += s
        heapq.heappush(h, s)
    return cost
```

与 1167 对照。C++ 用 `priority_queue<int, vector<int>, greater<int>>`。

### 附录式复习卡（仍属练习延伸，非独立 ##）

**卡 1 活动选择**：排序 end；`s>=last`；O(n log n)；交换 end。  
**卡 2 分数背包**：排序 v/w 降序；可分割；O(n log n)；交换密度。  
**卡 3 硬币**：贪心常错；322 min DP；反例 1,3,4,6。  
**卡 4 会议室**：排序 start；堆 end；O(n log n)。  
**卡 5 Huffman**：堆合并最小；O(n log n)。  
**卡 6 跳跃 55**：维护 far；O(n)。  
**卡 7 0-1**：不能密度贪心；knapsack_01。  
**卡 8 Kruskal**：边权升序；并查集；拟阵。  
**卡 9 Dijkstra**：非负权；dist 最小贪心。  
**卡 10 划界口诀**：能切密度，不能切 DP，硬币默认 DP。

**重复口述练习（建议录音 10 分钟）**

**专题总结段（练习延伸收尾，加长）**  
贪心算法在算法体系中的位置：它比暴力快、比 DP 省空间，但适用面窄。掌握本页意味着掌握「何时敢贪心」的判断，而不是掌握「所有题都贪心」。活动选择教会你排序键与扫描；分数背包教会你密度与交换；硬币反例教会你谦卑——回到 DP。Huffman 与会议室堆教会你数据结构配合贪心。面试最后一问常常是「还有更快做法吗」，若答案是「只有排序 O(n log n) 下界」，你可自信收尾。若答案是「需要 DP」，请主动说出状态定义，展示你会划界。本页与 Study 仓库形成闭环：笔记索引、可运行代码、站点 major 博文、校验脚本，四者一致才可 published。当前 manifest 为 draft，读者应以 strict 校验通过后的版本为权威发布。自学请打卡：第 1 周结束跑通双语言；第 2 周结束 AC 435 与 322 对比题；第 3 周结束能白板交换论证。教师备课可用基础篇分类树作板书大纲，用 240 与 [1,3,4] 作课堂例题，用十张复习卡作下课测验。企业内训可把「划界口诀」印在速查卡上：能切密度，不能切 DP，硬币默认 DP，区间不相交看 end，并行会议看 start 加堆。以上段落刻意重复核心口诀以强化记忆，属教学设计而非笔误。若你已能背诵口诀且手推无误，可只保留代码节与题表作日后参考，其余段落浏览标题即可。算法学习是长期项目，贪心一章练的是「结构识别」，请带着这一元目标完成练习与延伸，再进入下一专题。最后提醒：任何声称「万能贪心模板」的资料都应警惕；真正万能的是「先证再用」的习惯。把本页十张复习卡与 Study 双断言当作你工具箱里的扳手与螺丝刀——尺寸不对就换 DP，不要硬拧。若你向同学讲解本章，请用他们自己的话复述交换论证，你能听出对方是否真懂。教学相长，祝你在活动选择与分数背包之间，建立起对算法结构敬畏而务实的直觉。
