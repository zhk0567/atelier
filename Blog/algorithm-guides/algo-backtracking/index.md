# 算法 · 回溯（Backtracking）

## 导读

**回溯**是在**解空间树**上做深度优先搜索（DFS）的算法范式：每走一步做一次「选择」，进入下一层；若当前分支不可能得到合法解，或已收集完一种完整方案，则**撤销选择**（恢复现场）并尝试兄弟分支。与暴力枚举所有组合的区别在于：**剪枝**——在扩展前就判定约束不满足，从而跳过整棵子树。

Study 仓库 `backtracking/` 将回溯浓缩为四个可运行模板：**子集**、**组合**（从 `1..n` 选 `k` 个）、**全排列**、**N 皇后**（计数解的个数）。`notes.md` 用三句话概括：在解空间树上 DFS，满足约束则继续，否则剪枝；典型题即子集、组合、排列、N 皇后；复杂度**指数级**，剪枝可显著降低实际访问节点数。

本页在仓库笔记基础上系统展开：**决策树**如何画、**path / used / start** 三类状态如何选、**组合总和**（可重复选取、目标和）与**去重**（排序 + 同层跳过）的面试模板、复杂度上界与实际剪枝效果，以及一份可直接对照的**面试 checklist**。读完你应能：根据题面判断是「子集型 / 组合型 / 排列型 / 棋盘约束型」；写出统一的「选 → 递归 → 撤」骨架；知道何时排序去重、何时用 `start` 避免重复组合；并能用 PowerShell `-LiteralPath` 跑通 Study 的 Python 与 C++ 断言。

**与 DFS / DP / 贪心的分工**：图连通、岛屿、路径存在性常用普通 DFS/BFS，不必显式回溯撤销（除非路径要记录）。**计数方案数且重叠子问题**时考虑 DP（如爬楼梯、凑零钱最少枚数有时用 DP）。**活动选择、区间调度**可贪心。回溯适合「列出所有解」「判断是否存在一种构造」「计数所有合法排列/放置」且解空间呈**树形决策**、需要**尝试 + 回退**的题。

**面试中的回溯**：先口述解空间形状（每层选什么、深度多少），再写剪枝条件，最后给时间复杂度（常为指数，说明剪枝后实际节点更少）。仓库 `backtracking.py` 带 `__main__` 断言，建议本地跑通后再刷 LeetCode 78 / 46 / 51 / 39 / 40 / 90 等经典题。

## 预备知识

> **环境**：Python 3.10+；C++17，`g++`，`backtracking.cpp` 通过 `#include <alg_std.hpp>` 使用 `function`、`unordered_set` 等。

建议已掌握：

- **递归**：基线条件、递归调用、返回后执行顺序（用于理解「撤销」发生在递归返回之后）。
- **浅拷贝与 `path.copy()`**：收集答案时必须拷贝当前路径，否则 `out` 里全是同一列表引用。
- **集合 / 布尔数组**：N 皇后用 `set` 或数组 O(1) 判断列、主对角线 `r-c`、副对角线 `r+c` 是否占用。
- **排序**：含重复元素的组合/子集题，先排序再「同层跳过相同值」是去重标准写法。
- **复杂度直觉**：子集 `2^n`、排列 `n!`、组合 `C(n,k)`、N 皇后约 `O(n!)` 量级；剪枝只影响常数与隐藏因子，最坏仍指数。

**解空间树**：根为空解；第 `d` 层对应第 `d` 次决策。子集问题每层对 `nums[i]` 分支「要 / 不要」；组合问题每层从 `[start, n]` 选一个数；排列每层选一个未使用的下标；N 皇后每层在某一列放皇后。DFS 沿一条根到叶路径走到底，再回溯换兄弟节点。

**不变量**：回溯过程中 `path`（或棋盘行）始终表示「当前部分解」；进入递归前加入选择，返回前必须移除，保证兄弟分支看到相同起点状态。

## 基础篇

### 直觉与定义

回溯 = **DFS + 状态恢复**。三步口诀：**选择**（修改 `path` / `used` / 集合）、**探索**（递归下一层）、**撤销**（恢复现场以便尝试其他分支）。

**决策树（子集 `nums=[1,2,3]`）**：深度为 `n`，每层对应一个下标 `i` 的决策——「不选 `nums[i]`」走左子，「选」走右子。叶节点 `i==n` 时 `path` 即一个子集。共 `2^n` 个叶，对应全部子集。

```text
                    []
           /                    \
        不选1                   选1
        /    \                  /    \
     不选2  选2              不选2  选2
      ...    ...              ...    ...
```

**组合 `C(n,k)`**：深度为 `k`，每层从 `start..n` 选一个整数加入 `path`，下一层 `start` 设为 `x+1`，保证**递增顺序**，从而避免 `[2,1]` 与 `[1,2]` 重复。子集可看作「`k` 从 0 到 n 的所有组合」的并，但子集写法用选/不选更清晰。

**全排列**：深度 `n`，每层选一个**未使用**下标 `i`，`used[i]=True` 后递归，返回后 `used[i]=False`。第一层的三个分支分别是把 1、2、3 放在首位，各自展开 `(n-1)!` 子树，总计 `n!` 叶。

**N 皇后**：按行 `r` 决策，每行试列 `c`，用三个集合记录已占列、主对角、副对角；若冲突则**剪枝**不进入子树。叶节点 `r==n` 时计数加一。解的个数随 `n` 增长极快，小 `n` 用回溯教学，大 `n` 需更强算法或对称性（面试常只要求 `n≤9`）。

**组合总和（39/40，Study 未单独建文件）**：给定无重复正整数数组与 `target`，找出所有和为 `target` 的组合（39 可重复用同一数字，40 不可重复且数组可有重复值需去重）。建模为组合型回溯：`start` 控制顺序，剩余 `target - x` 为剪枝依据；`target<0` 时立即返回。

### 复杂度分析

| 问题 | 解空间规模 | 最坏时间 | 额外空间 |
|------|------------|----------|----------|
| 子集 | `2^n` | `O(n·2^n)` 含拷贝 path | `O(n)` 递归栈 |
| 组合 C(n,k) | `C(n,k)` | `O(k·C(n,k))` | `O(k)` |
| 全排列 | `n!` | `O(n·n!)` | `O(n)` used |
| N 皇后 | ≤ `n!`（强剪枝） | 指数，常写 `O(n!)` | `O(n)` 集合 |
| 组合总和 | 与分支因子相关 | 指数，剪枝 target | `O(n)` 栈 |

面试应说明：**最坏**是指数；**实际**访问节点数取决于剪枝（如 `target` 提前为负、N 皇后冲突检测）。输出所有解时，输出规模本身可能指数，算法无法低于输出量级。

**剪枝类型**：

1. **约束剪枝**：N 皇后列/对角冲突；组合总和 `remain < 0`。
2. **有序剪枝**：组合类 `start` 只向后选，避免重复组合。
3. **去重剪枝**：排序后 `i>start` 且 `candidates[i]==candidates[i-1]` 则 `continue`（同层不重复选相同值）。
4. **限界剪枝**：可重复选数字时，若已排序，下一层仍可从 `i` 开始（39）；不可重复则递归 `dfs(i+1)`（40）。

### 代码模板

**统一骨架（Python）**

```python
def backtrack(状态参数):
    if 到达终点:
        收集或更新答案
        return
    for 候选 in 候选列表:
        if 不满足约束:
            continue  # 剪枝
        做选择（修改 path / used / 集合）
        backtrack(下一状态)
        撤销选择
```

**子集（选 / 不选，Study `subsets`）**

```python
def dfs(i: int) -> None:
    if i == len(nums):
        out.append(path.copy())
        return
    dfs(i + 1)              # 不选 nums[i]
    path.append(nums[i])    # 选
    dfs(i + 1)
    path.pop()              # 撤销
```

**组合 C(n,k)（Study `combinations`）**

```python
def dfs(start: int) -> None:
    if len(path) == k:
        out.append(path.copy())
        return
    for x in range(start, n + 1):
        path.append(x)
        dfs(x + 1)
        path.pop()
```

**全排列（Study `permute`）**

```python
def dfs() -> None:
    if len(path) == len(nums):
        out.append(path.copy())
        return
    for i in range(len(nums)):
        if used[i]:
            continue
        used[i] = True
        path.append(nums[i])
        dfs()
        path.pop()
        used[i] = False
```

**N 皇后计数（Study `n_queens`）**

```python
def dfs(r: int) -> None:
    if r == n:
        ans += 1
        return
    for c in range(n):
        if c in cols or (r - c) in diag1 or (r + c) in diag2:
            continue
        cols.add(c); diag1.add(r - c); diag2.add(r + c)
        dfs(r + 1)
        cols.remove(c); diag1.remove(r - c); diag2.remove(r + c)
```

**组合总和 I（可重复，练习模板）**

```python
def dfs(start: int, remain: int) -> None:
    if remain == 0:
        out.append(path.copy())
        return
    if remain < 0:
        return
    for i in range(start, len(candidates)):
        path.append(candidates[i])
        dfs(i, remain - candidates[i])  # 仍可再选 candidates[i]
        path.pop()
```

**组合总和 II（不可重复 + 去重）**

```python
candidates.sort()

def dfs(start: int, remain: int) -> None:
    if remain == 0:
        out.append(path.copy())
        return
    for i in range(start, len(candidates)):
        if i > start and candidates[i] == candidates[i - 1]:
            continue
        if candidates[i] > remain:
            break  # 排序后可提前 break
        path.append(candidates[i])
        dfs(i + 1, remain - candidates[i])
        path.pop()
```

**含重复元素的子集（90）**：与子集相同 DFS，排序后 `if i > start_idx and nums[i]==nums[i-1]: continue` 放在 `for i in range(start_idx, n)` 的循环内，`start_idx` 为当前层起始下标（每层传 `start`，递归传 `i+1`）。

### 变体与技巧

- **子集 II（90）**：排序 + 同层去重；或先用 `Counter` 枚举每个值选 0..cnt 次（多重集合子集）。
- **组合总和 III（216）**：`1..9` 选 `k` 个和为 `n`，即 `combinations` 加 `remain` 剪枝。
- **分割回文（131）**：按起点 `i` 切分，`s[i:j]` 回文则 `dfs(j)`，路径存段。
- **单词搜索（79）**：网格 DFS + `visited`，回溯时取消访问标记。
- **括号生成（22）**：剩余左 `l`、右 `r`，`l<n` 可加 `(`，`r<l` 可加 `)`。
- **电话号码字母组合（17）**：按位映射字符串，深度为号码长度。
- **棋盘类**：数独、解数独、单词搜索 II（Trie+回溯）共用「尝试填数/字母 → 失败撤销」。
- **从 path 构造答案**：有的题只要求一种解，找到即 `return True` 向上传递，不必收集全部。
- **位运算排列**：`n≤10` 时可用 `mask` 表示已选集合，与状压 DP 不同，回溯仍枚举排列。

**`start` 与 `i` 的区别**：组合、组合总和用 `start` 保证**只向后看**，避免 `[2,3]` 与 `[3,2]`。子集用下标 `i` 同时表示「当前考虑元素」与「不选则 `i+1`」。排列不用 `start`，而用 `used` 因为**顺序有关**。

**去重黄金法则**：排序后，**在同一层 for 循环**中，若当前候选与上一个候选相同则跳过（`i > start` 或 `i > start_idx`）；**不要**在递归层简单 `if nums[i]==nums[i-1]: continue` 而不区分层，否则会误删合法解（如 `[1,1,2]` 需要两个 1 时，层内去重与跨层不同，90/40 标准写法是 `i>start`）。

### 易错点

1. **收集答案未拷贝**：`out.append(path)` 导致全是空或最后一态。
2. **组合未用 `start`**：产生重复组合如 `[2,1]`。
3. **排列忘记恢复 `used[i]`**：后续排列缺失元素。
4. **N 皇后对角线**：主对角 `r-c`、副对角 `r+c`，勿写成 `r+c` 与 `r-c` 混淆符号。
5. **去重条件写错**：应用 `i > start`（组合总和 II）或子集层的 `i > start_idx`，而非 `i > 0`。
6. **可重复与不可重复**：39 递归 `dfs(i, ...)`；40 必须 `dfs(i+1, ...)`。
7. **剪枝后仍 append**：`remain < 0` 应 `return` 而非先加入 path。
8. **空输入**：`subsets([])==[[]]`，`permute([])==[[]]`，面试提一句边界。
9. **C++ `path` 拷贝**：`out.push_back(path)` 在递归叶是值拷贝，与 Python `copy()` 对应。
10. **递归深度**：`n` 很大时 Python 可能栈溢出，竞赛可用迭代手写栈（面试少见）。

### 子集：手推决策树与复杂度

`nums=[1,2]` 时，Study 断言四种子集 `()`, `(1,)`, `(2,)`, `(1,2)`。DFS 从 `i=0` 开始：不选 1 走到不选 2 得 `[]`；不选 1 再选 2 得 `[2]`；选 1 后不选 2 得 `[1]`；选 1 再选 2 得 `[1,2]`。时间 `O(n·2^n)`：每个叶拷贝长度至多 `n` 的 path。空间递归栈 `O(n)`。

推广到 `n=3`：叶 8 个，对应 LeetCode 78。若元素有重复，须排序并在**同一层**跳过相同数字，否则 `[1,2]` 与 `[2,1]` 不会重复（子集无序），但 `[1,1]` 与 `[1,1]` 会因两条路径重复，90 题用 `i>start_idx and nums[i]==nums[i-1]` 去掉同层重复选取。

### 组合：C(4,2) 与 start 的作用

`combinations(4,2)` 输出六组：`[1,2],[1,3],[1,4],[2,3],[2,4],[3,4]`。`dfs(1)` 后第一层选 1，再 `dfs(2)` 选 2 满员；回溯后选 3、4 等同理。若错误写成 `for x in range(1, n+1)` 每层都从 1 开始，会得到 `[2,1]` 等重复。

组合总和 `candidates=[2,3,6,7], target=7`：路径 `[7]` 一层；`[2,2,3]` 需 39 的可重复 `dfs(i)` 而非 `dfs(i+1)`。排序后可在 `candidates[i]>remain` 时 `break`。

### 排列：used 数组与对称性

`permute([1,2,3])` 六排列。第一层三个分支，每个分支下两层各两个选择，共 `3×2×1`。47 含重复数字：排序后同层 `if i>0 and nums[i]==nums[i-1] and not used[i-1]: continue`（**used** 版本）或固定写法「前一个相同且未使用则跳过」以避免重复排列。

### N 皇后：剪枝与计数

`n=4` 时 Study 断言答案为 2。每行试列，集合判冲突；比检查 `board` 二维数组更省常数。51 要输出棋盘，在 `r==n` 时 `copy` 当前构造；52 只计数同 `n_queens`。面试可先说「按行放，列与两对角线 O(1) 判断」，再写代码。

### 组合总和 II 去重手推

`candidates=[1,1,2,5,6,7,10], target=8`：排序后有效组合 `[1,1,2],[1,7],[2,6]`。若 `i=1` 且 `candidates[1]==candidates[0]` 且 `i>start`，跳过，避免两条都选第二个 1 的重复路径；但 `start` 层仍可选第一个 1 再选第二个 1。

### 回溯与 DFS 图遍历

图 DFS 访问节点标记 `visited` 后通常**不撤销**（找连通块）；回溯在**同一层多个互斥选择**时必须撤销（皇后占列、排列 used、子集 path pop）。单词搜索在网格上**要撤销** `visited[i][j]`，因为不同路径可重用格。

### 面试 30 秒模板

「这是解空间树上的 DFS：每层做一种选择，用 path 记录部分解；不满足约束就剪枝；到底收集答案；返回前撤销选择。复杂度最坏指数，实际有剪枝。」

### 结语（基础篇）

回溯的核心不是背题号，而是**认决策树**：每层选什么、深度多少、如何避免重复、何时剪枝。Study 四函数是极简正确实现；组合总和与去重是面试必加的两种扩展，本页已在模板节给出与 39/40/90 一致的代码。请先在本地听到 `backtracking OK`，再进入题海；WA 时回仓库对拍函数行为，再检查是否忘记 `pop` 或 `copy`。

### 决策树与递归栈（深化）

递归调用栈深度等于决策深度：子集 `n`、组合 `k`、排列 `n`、皇后 `n`。每层栈帧保存局部变量与返回地址；**撤销**发生在 `dfs` 返回之后执行 `pop` / `used[i]=False` / `set.remove`。理解顺序可避免「在递归前 pop」的经典错误。

子集型也可改为迭代枚举 bitmask `0..2^n-1`，但面试与仓库统一用回溯写法。组合型 bitmask 选 `k` 个 1 亦可，但 `start` DFS 更直观。

### 剪枝效果估算（直觉）

N 皇后第一行 `n` 选 1，第二行至多 `n-1`（列冲突），实际远小于 `n^n`。组合总和排序后 `candidates[i]>remain` 可砍掉大量分支。理论最坏仍指数，面试诚实说明「最坏指数，剪枝改善平均情况」。

### 括号生成（22）状态设计

`dfs(l, r, path)`：`l<n` 时加 `(`；`r<l` 时加 `)`；`l==r==n` 收集。不需要撤销字符串时可传 `path` 拷贝，但标准仍用 `append` + `pop`。与组合无关，是**构造串**型回溯。

### 分割回文（131）

`dfs(i)` 从 `i` 切分，`j` 从 `i+1` 到 `n`，若 `s[i:j]` 回文则加入 path 并 `dfs(j)`，回溯 `pop`。深度至多 `n`，分支多时需要回文 DP 或 Manacher 预处理判断 O(1)。

### 单词搜索（79）

四方向 DFS，`visited` 标记后递归，返回前清除标记。与子集不同，**同一格可在不同路径重复使用**（不同分支），故必须撤销 visited。

### 复原 IP（93）

切四段，每段 1–3 位且数值 0–255 且无前导零（除单 0）。`dfs(start, part, path)`，`part==4` 且用完字符串时收集。

### 子集与组合的统一视角

子集 = 对每个元素 0/1；组合 = 恰好 `k` 个 1 且下标递增；排列 = `n` 个位置各填一个互异下标。三者都是**受限的 0/1 决策**，只是约束不同。

### 重复元素子集（90）再述

排序 `nums` 后 `dfs(start_idx)`，`for i in range(start_idx, n)`：若 `i>start_idx and nums[i]==nums[i-1]` 跳过；选 `nums[i]` 则 `dfs(i+1)`。注意 `start_idx` 每层独立，递归传入 `i+1` 而非 `start_idx+1` 的含义：下一层从 `i+1` 继续，同层去重靠 `i>start_idx`。

### 排列去重（47）再述

常见写法：排序后若 `nums[i]==nums[i-1]` 且 `not used[i-1]` 则 `continue`，保证相同数字的相对顺序只展开一次。与组合「同层跳过」逻辑相似但条件含 `used`。

### 输出所有解 vs 计数

### 子集专题课堂笔记（扩充）

子集问题是回溯入门的**标准模型**，因为每个元素只有两种决策，决策树深度恰好等于 `n`，结构完全对称。理解子集的关键是分清两层循环：外层递归参数 `i` 表示「当前考虑到第几个元素」，内层并不是 for 每个元素，而是**先递归不选、再递归选**两条分支。这种写法与「`for i in range(start, n)` 每个位置选或不选」的另一种子集写法等价，但 Study 仓库采用双 `dfs(i+1)` 结构，与 LeetCode 官方题解高度一致，建议作为默认模板背诵。

手算 `nums=[1,2,3]` 时，按时间顺序列出 DFS 访问的叶节点路径：不选1→不选2→不选3 得 `[]`；不选1→不选2→选3 得 `[3]`；不选1→选2→不选3 得 `[2]`；不选1→选2→选3 得 `[2,3]`；选1→不选2→不选3 得 `[1]`；选1→不选2→选3 得 `[1,3]`；选1→选2→不选3 得 `[1,2]`；选1→选2→选3 得 `[1,2,3]`。共 8 叶，与 `2^3` 一致。面试若要求**按字典序**输出子集，应先对 `nums` 排序，再使用本模板，输出自然按字典序递增。

子集与组合的区别在于：子集**不限制**选了几个元素，叶节点在 `i==n` 时一律收集；组合要求 `len(path)==k`，未到深度即使 `i==n` 也不收集。因此组合是子集上加**额外终止条件**与 **for 选数** 的特例，而不是另一套完全不同的算法。许多初学者把 78 与 77 当成两种算法，实际上 77 只是在 78 的骨架上把「不选分支」改成「必须从 start 选一个数」的 for 循环。

空间复杂度方面，递归栈深度 `O(n)`，输出列表存储 `O(n·2^n)` 个整数，这是问题本身规模决定的，不是实现浪费。Python 中 `path.copy()` 每次叶节点分配新列表，常数因子较大但 `n≤20` 面试数据可接受。若只问是否存在某子集和为目标，可在 `path` 和达到 target 时立即 `return True` 向上传递，不必收集全部，这是**搜索型**回溯而非**枚举型**回溯。

### 组合专题课堂笔记（扩充）

`combinations(n, k)` 生成的是 `1..n` 中选出 `k` 个数的所有**递增序列**，这是组合数学中的标准表示。递增性由 `start` 参数保证：当 path 末尾为 `x` 时，下一层只能从 `x+1` 开始尝试，因此不会出现 `[3,1]`。若题目给定的是任意数组 `candidates` 而非 `1..n`，仍需先排序（若有重复还要去重），再用 `dfs(start)` 其中 `start` 是 candidates 的下标，逻辑与仓库 `combinations` 完全一致，只是候选来自 `candidates[i]` 而非整数 `x`。

手算 `C(4,2)`：第一层选 1，第二层可选 2、3、4，得 `[1,2],[1,3],[1,4]`；回溯后第一层选 2，第二层可选 3、4，得 `[2,3],[2,4]`；再选 3 得 `[3,4]`。共 6 组，与 Study 断言 `len(combinations(4,2))==6` 一致。`k=0` 时特殊：循环 `for x in range(start, n+1)` 一次都不执行，在 `start=1` 时直接满足 `len(path)==0==k`，收集空列表，故 `combinations(3,0)==[[]]`。

组合总和 I（LeetCode 39）在组合框架上允许**重复使用同一候选**：递归调用 `dfs(i, remain - candidates[i])` 而不是 `dfs(i+1, ...)`，意味着下一层仍可从 `i` 选同一数字。若写成 `i+1` 则每个候选最多用一次，变成 40 题的无重复语义（在无重复数组上）。39 不要求 candidates 排序，但排序后可在 `candidates[i] > remain` 时 break，减少无效分支，是常用的**限界剪枝**。

组合总和 II（40）数组含重复且每个数字**最多用一次**：排序 + `dfs(i+1)` + 同层 `if i > start and candidates[i]==candidates[i-1]: continue`。手算 `candidates=[1,1,2,5,6,7,10], target=8`：有效解 `[1,1,2]`、`[1,7]`、`[2,6]`。注意 `[1,1,2]` 需要两个 1，同层去重只防止「同一层选两个相同的第二个 1」，不会阻止「第一层选第一个 1、第二层选第二个 1」这种跨层结构。错误写法 `if i>0 and candidates[i]==candidates[i-1]: continue` 会误杀 `[1,1,2]`，这是 40 题最高频 WA 原因之一。

216 题「组合总和 III」可视为 `combinations(9, k)` 加 `sum(path)==n` 剪枝：在 `dfs(start)` 中若 `sum(path)>n`  return，若 `len(path)==k` 且 sum 等于 n 收集，否则继续。与 39 的区别是固定 1..9 且无重复、无重复使用。

### 排列专题课堂笔记（扩充）

排列问题中**顺序有意义**，因此不能用 `start` 跳过「已考虑过的下标」那种组合技巧，必须用 `used[i]` 标记位置是否已在当前 path 中占用。`permute` 的 for 循环总是 `0..n-1`，跳过 `used[i]==True` 的下标。叶节点 `len(path)==n` 时收集。全排列数量为 `n!`，`n=10` 时约 362 万叶，Python 仍可在数秒内完成，但 `n=12` 以上列举全排列在面试中不现实，题目通常只问是否存在或下一个排列。

手算 `permute([1,2,3])` 的递归树：根空 path；第一层选 1、2、3 各一分支；以选 1 为例，第二层在 used 标记 0 后从 2、3 选，得 `[1,2]` 与 `[1,3]` 两条；第三层补全第三位。六个叶对应六个排列。47 题含重复元素：排序后，若 `nums[i]==nums[i-1]` 且 `used[i-1]==False`，说明前一个相同元素在本层尚未使用，当前层再选相同值会产生重复排列，应 `continue`。这条规则比组合去重多一个 `used` 条件，背诵时可记「排列去重看 used，组合去重看 start」。

下一个排列（31）可用算法迭代而非回溯，但面试偶尔要求「列出所有排列」仍用 `permute` 模板。字符串排列（无重复字符）与整数排列同构；有重复时用 47 规则。

### N 皇后专题课堂笔记（扩充）

N 皇后是**约束满足**型回溯：决策按行进行，每行恰好放一个皇后，列与两条对角线上不能有两个皇后。Study 的 `n_queens` 返回解的个数，使用三个 `set` 记录列占用、主对角 `r-c`、副对角 `r+c`。为什么主对角用 `r-c`？同一主对角线上任意两格 `(r1,c1)` 与 `(r2,c2)` 满足 `r1-c1==r2-c2`。副对角满足 `r1+c1==r2+c2`。冲突检测 O(1)，比扫描整行整列更高效。

`n=4` 时两解：一种为列顺序 `[1,3,0,2]`（0-based 列号），另一种对称。`n=8` 解数为 92，常用作自测。51 题输出棋盘需要维护 `board[r][c]='Q'` 或在 path 中用 `cols[r]=c` 记录列号，在叶节点将 path 转成字符串列表。52 题只计数，与仓库一致。面试写代码时先写 `dfs(r)` 框架，再填 for c 与三个集合的 add/remove，**撤销**必须与 add 对称，否则集合残留导致漏解或多解。

进阶：位运算压缩列与对角占用（`colMask`, `diag1Mask`, `diag2Mask`），适合竞赛 `n≤15`，不在 Study 仓库范围内，了解即可。面试不要求位运算版本。

### 组合总和与目标和（系统化）

39、40、216 三题构成「组合家族」的递进：39 可重复无重复元素；40 不可重复有重复元素需去重；216 固定 1..9 选 k 个和为 n。统一写法：维护 `remain`，进入递归前若 `remain<0` 剪枝，若 `remain==0` 收集。39 的 `dfs(i)` 表示仍可再选 `candidates[i]`；40 的 `dfs(i+1)` 表示 `candidates[i]` 已用掉不能再用；216 的 `dfs(x+1)` 与仓库 `combinations` 相同只是多了 remain 判断。

「和为 k 的子集个数」（非 LeetCode 主表但常问）可在子集回溯中维护 `remain`，不收集 path 而 `ans+=1`，或 DP 0-1 背包计数。列举所有和为 k 的子集则回溯收集。识别：要**所有方案**用回溯，只要**个数**且 n 小可回溯，n 大且仅 0-1 用 DP。

### 去重专题（系统化）

去重是回溯面试的**第二大难点**（第一是忘记撤销）。三类题去重写法对照：

| 题型 | 排序 | 跳过条件 | 递归参数 |
|------|------|----------|----------|
| 子集 II 90 | 是 | `i > start_idx` 且值相等 | `dfs(i+1)` |
| 组合 II 40 | 是 | `i > start` 且值相等 | `dfs(i+1)` |
| 排列 II 47 | 是 | 值相等且 `not used[i-1]` | `dfs()` |

核心思想：**在同一层 for 循环中，不要让两个相同候选出现在同一层的不同分支**（组合/子集），或**不要让相同元素在同一层被多次选为排列的第一个未用位置**（排列）。跨层仍可选相同值（如两个 1 组成 `[1,1,2]`）。

### 决策树与递归栈（再深化）

把回溯想成「深度优先遍历一棵隐式树」。每个内部节点对应一次选择；边代表选或不选、选哪个候选、放哪一列皇后。遍历顺序是**先走到底再回溯**，因此第一个完整解往往出现在左most 叶，最后一个解在右most 叶。若题目要求**字典序最小**的第一个解，应保证 for 循环候选顺序与题意一致（常先排序）。

递归栈帧保存：当前 `path`、`start`/`i`/`r`、局部集合副本（Python set 引用同一对象，add/remove 必须配对）。C++ 中 `function` 捕获引用 `[&]`，注意 `path` 与 `out` 生命周期在 `main` 内安全。不要在递归内对 `out` 做 clear。

### 剪枝分类与实例（再深化）

**可行性剪枝**：N 皇后列冲突；组合总和 `remain<0`；216 `sum(path)>n`。**最优性剪枝**（求最值时）：若当前部分解已不可能优于已知最优，则 return（回溯求排列字典序等少见）。**对称性剪枝**：N 皇后可利用对称性只搜一半再乘倍数，面试不写。**重复剪枝**：去重三类上表。

39 题排序后 `candidates[i]>remain` break 的依据：候选为正整数，后面更大更不可能。若有负候选则 break 失效，需改剪枝或改 DP。

### 网格与字符串回溯（概要）

79 单词搜索：从每个格作为起点 DFS，四向扩展，匹配失败则撤销 visited。212 单词搜索 II：Trie 存词表，DFS 时沿 Trie 走，减少重复前缀搜索。131 分割回文：切分点 `i`，试 `j>i`，子串回文则递归 `j`。93 复原 IP：四段，每段长度 1–3 且合法数值。22 括号：左括号数 `l`、右 `r`，`l<n` 加左，`r<l` 加右。17 电话：digits 每位映射字母串，深度为位数。这些题**共用**选择-递归-撤销，只是状态参数不同，练习时先写状态三元组再写代码。

### 回溯 vs 暴力 vs DP（对照表）

| 方法 | 适用 | 时间 | 输出 |
|------|------|------|------|
| 暴力枚举 | 极小 n | 指数 | 全部 |
| 回溯 | 列举解、约束强 | 指数，剪枝 | 全部或首个 |
| DP | 最优/计数，重叠子问题 | 多项或伪多项 | 数值 |

### 高频追问与应答

**Q：为什么不用 BFS？** A：要的是所有完整解的路径记录，DFS+回溯更自然；BFS 占内存且不便维护 path。**Q：能否排序优化子集？** A：子集 II 必须排序去重；无重复子集不必排序。**Q：n 很大怎么办？** A：题目保证 n 小或只问是否存在；否则换 DP/数学。**Q：C++ vector push_back path 会拷贝吗？** A：会，值语义，等价 Python copy。

### 手推：39 题 candidates=[2,3,6,7], target=7

排序已为升序。dfs(0,7)：试 2，remain=5，dfs(0,5) 再试 2，remain=3，再试 2，remain=1，试 2 超限；试 3 remain=0 收集 `[2,2,3]` 不对——更正：从 2,2,3 路径：选2 remain5，选2 remain3，选3 remain0 得 `[2,2,3]`；回溯试 6、7 超限或单 7 得 `[7]`。两条解与 LeetCode 一致。体现 `dfs(i)` 允许连续选 2。

### 手推：90 题 nums=[1,2,2]

排序后 dfs(0)：收集 `[]`；i=0 选1，dfs(1) 收集 `[1]`，再选2 得 `[1,2]`，再选第二个2 得 `[1,2,2]`；回溯 i=1 时 2==2 且 i>start 跳过？ i=1 为 start，不跳过，选第一个2；i=2 时 i>start 且 nums[2]==nums[1] 跳过第二个2 作为**同层**第二个分支，避免 `[1,2]` 与 `[1,2]` 重复路径。最终子集含 `[],[1],[2],[1,2],[2,2],[1,2,2]` 等，需完整 DFS 树验证无漏无重。

### 手推：47 题 nums=[1,1,2]

排序后，第一层选 index0 的 1，第二层只能选 2，得 `[1,2]`；第一层若试图选 index1 的 1，因 `nums[1]==nums[0]` 且 `not used[0]` 跳过。最终排列 `[1,2,2,1]` 等去重后 3 个。与 `permute` 加 47 规则一致。

### 理论背景（简短）

回溯属于**深度优先搜索**在组合优化中的特例；完备性：若解存在且剪枝正确，DFS 必能找到；最优性不保证（除非额外剪枝证明）。与分支限界、A* 相比，回溯实现短、适合面试。工业界配置生成、测试用例组合常用回溯+强剪枝。

### 与图论 DFS 的再次对比

`algo-graph-traversal` 中访问节点标记后通常不撤销，因为只需知道可达性；回溯在**同一问题需要尝试多种互斥选择并恢复**时使用。岛屿数量、拓扑排序不是回溯主战场；**路径列举**（所有路径）在图上可用回溯，边权版本注意环检测。

### 位运算枚举子集（补充）

对 `n≤20`，可用 `mask` 从 0 到 `2^n-1`，内层检查每位是否选 `nums[i]`，无需显式递归。面试仍推荐回溯写法因可中途剪枝。状压 DP 的 mask 与回溯子集枚举不同：DP 求最优/计数，回溯求构造列表。

### 子集问题逐层 DFS 时间线（长示例）

以下用 `nums=[1,2]` 按调用栈顺序叙述，帮助建立「先深后广」直觉。调用 `dfs(0)`：不选 1 进入 `dfs(1)`；不选 2 进入 `dfs(2)`，`i==len` 收集 `[]` 返回；回到 `dfs(1)` 选 2，`path=[2]`，进入 `dfs(2)` 收集 `[2]` 返回；pop 2 回到 `dfs(0)` 选 1，`path=[1]`，进入 `dfs(1)` 不选 2 收集 `[1]`；选 2 得 `[1,2]`。四次收集与 Study 断言四元组一致。扩展 `nums=[1,2,3]` 时叶数为 8，每层分叉 2，总节点约 `2^{n+1}-1` 量级。面试手画时只需画三层二叉树，标出叶 path 即可。

### 组合问题逐层 DFS 时间线（C(4,2)）

`dfs(1)` path 空：选 1，`dfs(2)` 选 2 满员收集 `[1,2]`；回溯选 3 得 `[1,3]`；选 4 得 `[1,4]`；回溯到 `dfs(1)` 选 2，`dfs(3)` 选 3 得 `[2,3]`，选 4 得 `[2,4]`；再选 3，`dfs(4)` 得 `[3,4]`。六次收集。注意**从未出现** `[2,1]` 因为 `start` 递增。若题目要求组合来自无序数组 `candidates=[2,1,3]`，须先排序为 `[1,2,3]` 再用下标 `start`，否则 `[2,1]` 会出现。

### 排列问题逐层 DFS 时间线（三元素）

第一层三个分支分别置 `used[0/1/2]`。以选 0 为例：path `[1]`，第二层可选 1 或 2（下标），第三层补全。六叶对应 123、132、213、231、312、321。47 题在第二层若两元素值相同且前一个未使用则剪枝，树宽减小。理解「层」的概念对去重至关重要：层 = 当前 `path.length` 固定时 for 循环的迭代，不是递归深度本身，但二者在排列中一致。

### N 皇后四层手推（n=4）

第 0 行试列 0、1、2、3；列 0 放后第 1 行试列 2、3 等（列 1 冲突）。完整枚举需耐心，最终得 2 解。面试可只画部分树说明「列冲突剪枝」，不必画满。52 题 n=8 用程序验证 92，手算不现实。

### 39 题完整分支叙述（target=7）

candidates 排序 `[2,3,6,7]`。根 remain=7：选 2 remain5，再选 2 remain3，再选 3 remain0 收集；回溯 remain3 选 6 超；选 7 超；回溯 remain5 选 3 remain2 再选 2 得 `[2,3,2]` 无效若不允许重复顺序？ 39 允许重复数字可 `[2,3,2]` 若和为 7？ 2+3+2=7 是另一解，dfs(i) 允许。还有单 7。多解列举练剪枝。

### 40 题与 39 题并列对照（长文）

39：元素互异，可重复使用同一元素，递归 `dfs(i)`。40：元素可重复但每个**下标**只能用一次，递归 `dfs(i+1)`，同层去重。混用两种规则会导致 39 少解或 40 多解。写题前先读英文 "each candidate may only be used once" 与 "unlimited number of times"。216 是 40 的特例：候选固定 1..9 且无重复，只需 `start` 与 `k`、`n` 判断。

### 90 题与子集模板的唯一差异

`subsets_with_dup` 在 **进入 dfs 前** 就 `out.append(path.copy())`（所有节点收集），而 78 只在叶收集。这是子集 II 标准写法：排序后中间节点也合法，避免漏解。78 的叶收集与 90 的中间收集不要混用。记忆：有重复子集 → 排序 + 每层开头收集 + `i>start` 跳过。

### 47 题与 permute 的唯一差异

仅 for 循环头部多一条去重 if。其余 `used`、`path`、`dfs()` 不变。改题时复制 permute 再插入一行，减少手写错误。

1. 将 `ans` 改为 `out: List[List[str]]`。2. 增加 `cols` 数组或 `board`。3. 叶节点构造 n 行字符串加入 out。4. 仍用三个 set 剪枝。5. 返回 out。52 保持计数即可。

### 22 括号生成 DFS 详解

`n=3` 时合法串五个：((()))、(()())、(())()、()(())、()()()。DFS：l 表示已放左括号数，r 已放右括号数，当 l<n 放左；当 r<l 放右；l==r==n 收集。无效如 )( 永远不会出现因为 r<l 才放右。深度 2n，叶 卡特兰数 C_n，n=3 时 5。与回溯模板一致。

### 17 电话号码字母组合

digits 长度 m，深度 m，每层映射 3–4 个字母分支，总分支积。空 digits 返回 `[]` 特例。状态为当前构建字符串或 path 列表。

### 79 单词搜索细节

对每个格 (i,j) 作起点 DFS。函数签名 `dfs(i,j,idx)` idx 为匹配到 word 第几位。越界或字符不等 return False。成功则 mark visited，四向递归，失败撤销 visited。找到一个可短路 return True。

### 131 回文分割详解

`dfs(i)` 表示从 i 开始切。`j` 从 i+1 到 n，若 `s[i:j]` 回文则 push 段，`dfs(j)`，pop。收集在 i==n 时。回文判断可双指针 O(n)，总复杂度近似 O(n·2^n) 最坏。

### 93 复原 IP 详解

`dfs(start, part, path)` part 为已用段数 0..4。每步取 1–3 字符转数字，前导零只允许单 0。part==4 且 start==len(s) 收集。剪枝：剩余字符过多或过少无法分成四段。

### 473 火柴拼正方形（了解）

排序火柴从长到短，四条边 DFS 填，和为 sum/4。超时则剪枝：当前边失败且是边内第一个尝试的火柴可 break。回溯型但剪枝强。

### 698 分割等和子集（了解）

### 526 优美排列（了解）

### 专题复习：决策树的数学刻画

解空间树可视为一棵深度为 d 的 k 叉树（k 随层变化），叶节点对应完整解，内部节点对应部分解。回溯 DFS 访问顺序是**前序遍历**到叶，再回溯兄弟。节点访问次数上界为 O(节点数)，节点数在子集问题为 `2^{n+1}-1` 量级。剪枝相当于砍掉子树，不改变正确性，只减少访问。证明剪枝正确：被砍子树内任一叶均不满足约束，故不可能出现在答案集中。

### 专题复习：子集与幂集

幂集 P(S) 的大小为 `2^{|S|}`，子集回溯正是在枚举幂集。若 S 含重复元素，幂集元素个数仍按**下标**区分，去重后集合意义下的子集个数减少，故 90 题需同层去重。数学符号与代码 `i` 下标一致，避免集合论与实现脱节。

### 专题复习：排列与双射

n 个**互异**元素的排列与 n 位互异下标序列一一对应，总数 n!。47 题元素有重，排列数公式含重复因子 `n!/(n1!n2!...)`，代码去重实现该计数，不必先算公式再枚举。

### 专题复习：组合与二项式系数

`C(n,k)` 即二项式系数，组合回溯枚举所有 k 组合。帕斯卡三角递推是 DP 视角，回溯视角是**直接构造**每个 k 元组。216 题附加和为 n 约束，相当于在 C(9,k) 中筛选。

### 专题复习：皇后与约束传播

列集合、对角集合是**约束传播**的极简形式：放皇后瞬间禁止同列同对角后续行选择。更强 CSP 用前向检查，面试 set 版足够。n=14 解数为 365596469 可参考 OEIS，程序秒级。

### 39 题所有解列举（文字版）

target=7，candidates=[2,3,6,7]：解1 `[7]`；解2 `[2,2,3]`（2+2+3=7）；无 `[3,3,1]` 因无 1。若 candidates 含 1 则分支更多。理解「可重复」即同一数字可占多个 path 位置。

### 40 题无效路径示例

candidates=[1,1,2], target=3：正解 `[1,2]`（用两个不同下标的 1 与 2？ 1+2=3 只用各一次）实际 [1,2] 用第一个1和2。若同层选两个第二个1会重复路径被跳过。target=2 解 `[1,1]` 需两个 1，跨层选取。

## Python 实现

以下为 `F:\Study\Algorithm\python\algorithms\backtracking\backtracking.py` 全文，与仓库一致，含 `__main__` 断言。

```python
"""回溯：子集、组合、排列、N 皇后。"""

from __future__ import annotations


def subsets(nums: list[int]) -> list[list[int]]:
    out: list[list[int]] = []
    path: list[int] = []

    def dfs(i: int) -> None:
        if i == len(nums):
            out.append(path.copy())
            return
        dfs(i + 1)
        path.append(nums[i])
        dfs(i + 1)
        path.pop()

    dfs(0)
    return out


def combinations(n: int, k: int) -> list[list[int]]:
    out: list[list[int]] = []
    path: list[int] = []

    def dfs(start: int) -> None:
        if len(path) == k:
            out.append(path.copy())
            return
        for x in range(start, n + 1):
            path.append(x)
            dfs(x + 1)
            path.pop()

    dfs(1)
    return out


def permute(nums: list[int]) -> list[list[int]]:
    out: list[list[int]] = []
    used = [False] * len(nums)
    path: list[int] = []

    def dfs() -> None:
        if len(path) == len(nums):
            out.append(path.copy())
            return
        for i in range(len(nums)):
            if used[i]:
                continue
            used[i] = True
            path.append(nums[i])
            dfs()
            path.pop()
            used[i] = False

    dfs()
    return out


def n_queens(n: int) -> int:
    """返回不同解的个数（n 小）。"""
    cols = set()
    diag1 = set()
    diag2 = set()
    ans = 0

    def dfs(r: int) -> None:
        nonlocal ans
        if r == n:
            ans += 1
            return
        for c in range(n):
            if c in cols or (r - c) in diag1 or (r + c) in diag2:
                continue
            cols.add(c)
            diag1.add(r - c)
            diag2.add(r + c)
            dfs(r + 1)
            cols.remove(c)
            diag1.remove(r - c)
            diag2.remove(r + c)

    dfs(0)
    return ans


if __name__ == "__main__":
    assert len(subsets([1, 2])) == 4 and {tuple(s) for s in subsets([1, 2])} == {(), (1,), (2,), (1, 2)}
    assert len(combinations(4, 2)) == 6
    perms = {tuple(p) for p in permute([1, 2, 3])}
    assert len(perms) == 6
    assert n_queens(4) == 2
    assert subsets([]) == [[]]
    assert combinations(3, 0) == [[]]
    assert permute([]) == [[]]
    print("backtracking OK")
```

**函数说明**

- `subsets`：对 `nums[i]` 先跳过后选用，`i` 从 0 到 `len(nums)`，叶节点收集子集。空数组返回 `[[]]`。
- `combinations`：从 `1..n` 中选 `k` 个递增序列；`k=0` 时 `dfs(1)` 直接收集空 path。
- `permute`：全排列，`used` 防重复选同一位置；空数组返回 `[[]]`。
- `n_queens`：返回解的**个数**，不构造棋盘；`n=4` 时为 2。

```python
def combination_sum(candidates: list[int], target: int) -> list[list[int]]:
    out: list[list[int]] = []
    path: list[int] = []
    candidates.sort()

    def dfs(start: int, remain: int) -> None:
        if remain == 0:
            out.append(path.copy())
            return
        if remain < 0:
            return
        for i in range(start, len(candidates)):
            if candidates[i] > remain:
                break
            path.append(candidates[i])
            dfs(i, remain - candidates[i])
            path.pop()

    dfs(0, target)
    return out
```

```python
def subsets_with_dup(nums: list[int]) -> list[list[int]]:
    nums.sort()
    out: list[list[int]] = []
    path: list[int] = []

    def dfs(start: int) -> None:
        out.append(path.copy())
        for i in range(start, len(nums)):
            if i > start and nums[i] == nums[i - 1]:
                continue
            path.append(nums[i])
            dfs(i + 1)
            path.pop()
```

## C++ 实现

以下为 `F:\Study\Algorithm\cpp\algorithms\backtracking\backtracking.cpp` 全文，与仓库一致。

```cpp
#include <alg_std.hpp>
#include <cassert>
using namespace std;

vector<vector<int>> subsets(vector<int> nums) {
    vector<vector<int>> out;
    vector<int> path;
    function<void(int)> dfs = [&](int i) {
        if (i == (int)nums.size()) {
            out.push_back(path);
            return;
        }
        dfs(i + 1);
        path.push_back(nums[i]);
        dfs(i + 1);
        path.pop_back();
    };
    dfs(0);
    return out;
}

vector<vector<int>> combinations(int n, int k) {
    vector<vector<int>> out;
    vector<int> path;
    function<void(int)> dfs = [&](int start) {
        if ((int)path.size() == k) {
            out.push_back(path);
            return;
        }
        for (int x = start; x <= n; ++x) {
            path.push_back(x);
            dfs(x + 1);
            path.pop_back();
        }
    };
    dfs(1);
    return out;
}

vector<vector<int>> permute(vector<int> nums) {
    vector<vector<int>> out;
    vector<int> path;
    vector<char> used(nums.size(), 0);
    function<void()> dfs = [&]() {
        if ((int)path.size() == (int)nums.size()) {
            out.push_back(path);
            return;
        }
        for (int i = 0; i < (int)nums.size(); ++i) {
            if (used[i]) continue;
            used[i] = 1;
            path.push_back(nums[i]);
            dfs();
            path.pop_back();
            used[i] = 0;
        }
    };
    dfs();
    return out;
}

int n_queens(int n) {
    unordered_set<int> cols, d1, d2;
    int ans = 0;
    function<void(int)> dfs = [&](int r) {
        if (r == n) {
            ++ans;
            return;
        }
        for (int c = 0; c < n; ++c) {
            if (cols.count(c) || d1.count(r - c) || d2.count(r + c)) continue;
            cols.insert(c);
            d1.insert(r - c);
            d2.insert(r + c);
            dfs(r + 1);
            cols.erase(c);
            d1.erase(r - c);
            d2.erase(r + c);
        }
    };
    dfs(0);
    return ans;
}

int main() {
    auto s = subsets({1, 2});
    assert(s.size() == 4);
    auto c = combinations(4, 2);
    assert(c.size() == 6);
    auto p = permute({1, 2, 3});
    assert(p.size() == 6);
    assert(n_queens(4) == 2);
    cout << "backtracking OK" << endl;
    return 0;
}
```

|----|--------|-----|

| 皇后对角 | `r-c`, `r+c` | `d1`, `d2` 同名集合 |
