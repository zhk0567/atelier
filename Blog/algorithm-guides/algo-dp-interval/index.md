# 算法 · 区间动态规划（Interval DP）

## 导读

**区间 DP** 处理的对象是闭区间 `[i, j]`：在数组、矩阵链、石子合并、括号序列等场景里，最优解往往由「把区间在某处切开」的两段子区间组合而成。与 `algo-dp-linear` 沿前缀或时间推进、`algo-dp-knapsack` 沿容量维滚动不同，区间 DP 的典型填表顺序是 **按区间长度 `len` 从小到大**，再枚举左端点 `i`、右端点 `j = i + len - 1`，在 `[i, j)` 内枚举切分点 `k`。

Study 仓库 `interval/` 以 **矩阵链乘法**（Matrix Chain Multiplication）为锚点实现 `matrix_chain_order`：给定维度数组 `p`，`n = len(p)-1` 个矩阵，求最少标量乘法次数。该模型是 OI 与面试中区间 DP 的「标准题」，同一套三层循环可迁移到石子合并、戳气球、回文划分等题，只需改代价函数与边界。

本页在 `notes.md` 骨架上扩写：区间状态的含义、矩阵链代价公式 `p[i]*p[k+1]*p[j+1]` 从何而来、与分治递推的关系、**石子合并**与 **312 戳气球** 的建模对照，以及为何复杂度通常是 **O(n³)**。读完你应能默写 Study 函数、解释 `length` 外层循环的必要性，并在 Python 与 C++ 中对拍断言。

**在 DP 家族中的位置**：`algorithms/dynamic_programming/interval` 与 `linear`、`knapsack`、`tree`、`bitmask` 并列；总览见 `algo-dynamic-programming`。Hot 100 中 312、1000 等与区间相关，本页聚焦 **按长度扩区间 + 枚举切分点** 这一实现核心。

**面试沟通顺序**：30 秒说明「状态是区间 `[i,j]`，答案由切分点合并」→ 写出三层循环方向（`len` 升序）→ 报 `O(n³)` 时间与 `O(n²)` 空间 → 举一道矩阵链或石子合并手推小样例。

**为何值得系统学**：区间 DP 的 WA 高度集中在三类：循环长度顺序写反、矩阵链 `p` 下标 off-by-one、石子合并漏加整段 `sum(i,j)`。修一处往往全局通过。Study 只实现矩阵链，但三层循环骨架可迁移多题，与 `algo-dp-knapsack` 的「方向敏感」同属一类记忆点。

**与线性 DP 对照**：LCS 的 `dp[i][j]` 也定义在二维下标上，但填表按 `i+j` 或 `i` 递增，因为依赖的是前缀而非「更短区间」。区间 DP 必须按 **长度** 扩，因为 `dp[i][j]` 只依赖严格更短的子区间。混淆这两种填表顺序是常见错误。

**与背包对照**：背包的维度是「资源容量」；区间的维度是「左右端点」。题面若写「合并连续段」「括号化」「最后戳破」，不要用背包逆序容量循环。Hot 100 中 312 与 416（子集和）易混：416 是 0-1 背包可达，312 是区间最后操作。

阅读本专题前，建议已具备：

- **动态规划基础**：最优子结构、填表顺序、无后效性；若生疏可先读 `algo-dp-linear`。
- **矩阵乘法维度**：`A` 为 `a×b`，`B` 为 `b×c`，乘积 `AB` 为 `a×c`，标量乘法次数为 `a·b·c`。
- **嵌套循环**：理解 `for length`、`for i`、`for k` 如何保证子区间先于大区间被计算。
- **大数**：矩阵链代价可能很大，C++ 用 `long long`，Python 整数任意精度；初始化「不可达」常用 `10**18` 或 `LLONG_MAX/4`。

**填表口诀**：外层 **区间长度** 从 2 到 n；中层 **左端点** `i`；内层 **切分点** `k ∈ [i, j-1]`。单点区间 `[i,i]` 代价为 0（或题面定义的基态）。

**重叠子问题**：计算 `dp[i][j]` 时子区间 `[i,k]`、`[k+1,j]` 在更大区间中重复出现；填表每个子区间只求一次最优。记忆化 `f(i,j)` 与迭代等价，长 n 建议迭代避免栈溢出。

**最优子结构**：若最优括号化在 `[i,j]` 的最后一步合并左右子链，则左右子链各自也必须最优，否则可替换为更优子结构得到更优整体，矛盾。故枚举切分点 k 取 min 正确。

**无后效性**：一旦确定 `[i,j]` 内最优合并方式，与区间外无关，只依赖子区间最优值与跨越代价。这与线性 DP 前缀无后效类似，但维度是区间而非单下标。

**工具链**：VS Code/Cursor 同时打开 Study python 与 cpp 目录；改转移时三处同步或至少双语言断言通过。PowerShell 7 与 5.x 均支持 `-LiteralPath`；路径含空格勿省略引号。

**从暴力到 DP**：暴力枚举所有切分点树是 Catalan 级；DP 把每个 `[i,j]` 最优存表，复杂度 O(n³)。面试先口述暴力再写 DP 体现思维。

**与树形 DP**：树形在节点子树上合并；区间在 **线性序列** 的连续段上合并。题面是树用 `algo-dp-tree`，是数组连续段用本页。

**工程角度**：矩阵链在科学计算库选乘法顺序有现实意义；石子合并是经典 OI 模型。代码短、边界集中，适合 Study 单函数断言学习。

## 基础篇

### 直觉与定义

**问题抽象**

有一串对象下标 `0..n-1`（矩阵、石子、气球等），对 **连续子段** `[i, j]` 定义代价或收益，整体答案常是 `dp[0][n-1]`。区间 DP 的核心是：**大区间最优 = 枚举切分点 k，合并左段 `[i,k]` 与右段 `[k+1,j]` 的最优，再加上「跨过 k 的合并代价」**。

**矩阵链乘法（Study 实现）**

`n` 个矩阵 `A1..An`，`Ai` 维度为 `p[i-1] × p[i]`（Study 用长度 `n+1` 的数组 `p`，`Ai` 为 `p[i]×p[i+1]`）。求括号化方案使标量乘法次数最少。

- **状态**：`dp[i][j]` = 计算矩阵 `Ai+1..Aj`（下标从 1 计）所需最少乘法次数；代码里 `i,j` 为 0-based 闭区间，对应 `Ai+1` 到 `Aj` 共 `j-i+1` 个矩阵。
- **边界**：`dp[i][i] = 0`（一个矩阵无需乘）。
- **转移**：在 `i..j` 之间选切分点 `k`，先分别乘好左链与右链，再乘左右结果：

  `dp[i][j] = min over k ( dp[i][k] + dp[k+1][j] + p[i]*p[k+1]*p[j+1] )`

  最后一项：左积维度 `p[i]×p[k+1]`，右积维度 `p[k+1]×p[j+1]`，相乘代价 `p[i]*p[k+1]*p[j+1]`。

**手推样例（Study 断言）**

`p = [10, 20, 30, 40]`，三个矩阵 10×20、20×30、30×40。最优先算前两个得 10×20×30=6000，再与第三个乘 10×30×40=12000，合计 **18000**。先算后两个再与第一个乘同样为 18000。`matrix_chain_order` 断言 `6000+12000`。

**石子合并（建模对照）**

`n` 堆石子排成一行，每次合并相邻两堆，代价为两堆石子数之和，求合并成一堆的最小总代价。常先做 **前缀和** `sum[i..j]`，令 `dp[i][j]` 为合并 `[i,j]` 的最小代价：

`dp[i][j] = min_{i<=k<j} ( dp[i][k] + dp[k+1][j] + sum(i,j) )`

结构与矩阵链相同，只是合并代价从 `p[i]*p[k+1]*p[j+1]` 换成区间石子总和。环形石子可先断环复制一倍或枚举断点。

**戳气球 312（思路）**

戳破气球 `i` 时，左右邻居相乘得分；用「最后戳破 `k`」转化区间：`dp[i][j]` 表示开区间 `(i,j)` 内剩余气球的最大分，枚举最后戳的 `k`，两侧独立。边界需虚拟气球 `i-1`、`j+1` 值为 1。仍是区间 + 切分点，代价函数不同。

**与分治的关系**

矩阵链乘法朴素分治也是枚举切分点，区间 DP 把子问题存表避免重复计算，复杂度从指数降到 O(n³)。

### 复杂度分析

| 项目 | 说明 |
|------|------|
| 状态数 | `O(n²)` 个区间 `[i,j]` |
| 转移 | 每个区间枚举 `O(n)` 个 `k` |
| 时间 | `O(n³)` |
| 空间 | `O(n²)` 二维表；可滚动优化视题而定，矩阵链通常保留全表 |

`n ≤ 400~500` 时 O(n³) 在 C++ 常可接受；Python 需注意常数。若题目带额外维度（如石子条数 + 合并次数），状态会膨胀，需单独分析。

**与暴力对比**：切分点组合等价 Catalan 级，暴力指数级；DP 合并重叠子区间。

### 代码模板

以下与 Study 仓库逻辑一致。

```python
def matrix_chain_order(p: list[int]) -> int:
    n = len(p) - 1
    if n <= 0:
        return 0
    dp = [[0] * n for _ in range(n)]
    for length in range(2, n + 1):
        for i in range(n - length + 1):
            j = i + length - 1
            dp[i][j] = 10**18
            for k in range(i, j):
                cost = dp[i][k] + dp[k + 1][j] + p[i] * p[k + 1] * p[j + 1]
                dp[i][j] = min(dp[i][j], cost)
    return dp[0][n - 1]
```

**石子合并模板（非仓库代码，练习用）**

```python
def stone_merge(stones: list[int]) -> int:
    n = len(stones)
    if n <= 1:
        return 0
    pre = [0]
    for x in stones:
        pre.append(pre[-1] + x)
    def seg(i, j):
        return pre[j + 1] - pre[i]
    dp = [[0] * n for _ in range(n)]
    for length in range(2, n + 1):
        for i in range(n - length + 1):
            j = i + length - 1
            dp[i][j] = 10**18
            for k in range(i, j):
                dp[i][j] = min(dp[i][j], dp[i][k] + dp[k + 1][j] + seg(i, j))
    return dp[0][n - 1]
```

### 变体与技巧

**回文划分 / 布尔区间**

`dp[i][j]` 表示 `s[i..j]` 是否回文，按长度扩展：`dp[i][j] = (s[i]==s[j]) and dp[i+1][j-1]`，O(n²)，常作区间 DP 入门。

**括号匹配计数**

类似回文，转移看 `s[i]` 与 `s[j]` 是否成对及内部是否合法，方案数可对 `10^9+7` 取模。

**区间 gcd / 区间最值**

若只问静态区间最值，用 ST 或线段树而非 DP；区间 DP 用于 **合并代价依赖区间整体** 的问题。

**四边形不等式优化**

部分区间 DP（如石子合并）满足单调最优切分点，可优化到 O(n²)，竞赛向；面试提到即可。

### 易错点

1. **循环顺序**：必须先算短区间再长区间；`i`、`j` 由 `length` 推导，勿写反。
2. **代价下标**：矩阵链合并项是 `p[i]*p[k+1]*p[j+1]`，与 `p` 长度 `n+1` 对齐；`j+1` 勿写成 `j`。
3. **单矩阵**：`n<=0` 或 `len(p)==1` 应返回 0，Study 已处理。
4. **初始化**：`dp[i][j]` 在 `length>=2` 时先置无穷再 `min`，勿用 0 当初值。
5. **石子合并**：合并代价是 **整个** `[i,j]` 的石子总和，不是只算 `k` 两侧。
6. **戳气球**：开区间与闭区间定义不同，勿直接套矩阵链下标。
7. **C++ 溢出**：乘积用 `long long`，初值 `LLONG_MAX/4`。
8. **环形**：需断环或倍数组，答案取 min over 断点。

**1000 石子合并（详解）**：给定数组 `stones`，每次合并相邻两堆，新堆重量为两堆之和，代价为新堆重量，求合并成一堆最小总代价。若可合并任意两堆（非仅相邻）则是另一问题；LeetCode 1000 是 **相邻** 合并，故是区间 DP。`pre` 前缀和后 `seg(i,j)` 为区间和。转移与矩阵链同形，代价项换为 `seg(i,j)`。环形版本枚举断点或复制数组长度 `2n`。

**312 戳气球（详解）**：数组 `nums` 表示气球编号，戳破 `i` 得分 `nums[i-1]*nums[i]*nums[i+1]`（虚拟边界为 1）。定义 `dp[i][j]` 为开区间 `(i,j)` 内（不含 i,j 边界气球）能获得的最大硬币。枚举 **最后** 戳破的气球 `k`：`dp[i][j]=max_{i<k<j}(dp[i][k]+dp[k][j]+nums[i]*nums[k]*nums[j])`。注意下标是开区间，边界虚拟 1。与矩阵链的「闭区间 + 加法代价」不同，但 **按长度扩区间 + 枚举切分点** 相同。

**516 最长回文子序列（与区间关系）**：可定义 `dp[i][j]` 为 `s[i..j]` 的最长回文子序列长度。若 `s[i]==s[j]` 则 `dp[i][j]=dp[i+1][j-1]+2`，否则 `dp[i][j]=max(dp[i+1][j], dp[i][j-1])`。按长度扩展，仍是区间型状态，但转移不是「枚举切分点合并」，而是 **依赖更短区间 + 缩边界**。识别：若合并代价是「整段 sum」或「最后乘在一起」，用切分型；若依赖两端字符匹配，用缩边型。

**1039 三角剖分（思路）**：多边形顶点剖分成三角形，代价与矩阵链类似，可定义 `dp[i][j]` 为顶点 i..j 多边形剖分最小代价，枚举切分点 k，代价含三角形边权乘积。实现时查阅题面边权定义，循环骨架不变。

**546 移除盒子（了解）**：三维状态 `dp[i][j][k]` 表示区间 `[i,j]` 且 `j` 右侧连续 k 个与 `j` 同色盒子的最大分。是区间 DP 高阶扩展，面试极少要求完整实现，知道「区间 + 额外维度」即可。

**记忆化写法（面试口述）**：`@lru_cache def f(i,j):` 枚举 k 取 min，与迭代三层循环等价。递归深度 O(n)，长链可能栈溢出，生产与竞赛更常用迭代填表。

**输出括号方案（Follow-up）**：维护 `split[i][j]=最优 k`，递归打印：若 `i==j` 输出单个矩阵；否则打印 `( f(i,k) f(k+1,j) )`。复杂度 O(n) 回溯。

**环形石子（Follow-up）**：复制 `stones` 为长度 `2n-1` 的数组，枚举断点 `start`，对 `[start, start+n-1]` 跑区间 DP，答案取 min。复杂度 O(n³) 乘 n 为 O(n⁴) 朴素；优化可合并。

**四边形不等式（竞赛）**：石子合并满足最优切分点 `k` 随 `i,j` 单调，可把内层 k 优化为 O(1) 均摊，总 O(n²)。面试不必实现，答「可优化到 O(n²)」加分。

**与 Catalan 数**：n 个矩阵括号化方案数为 Catalan(n)，暴力枚举切分点指数级；DP 把每个子区间最优只算一次。理解重叠子问题：子区间 `[i,k]` 与 `[k+1,j]` 在多个大区间中重复出现。

**伪多项式提醒**：区间 DP 时间是 n 的多项式，与背包的「容量数值伪多项式」不同；瓶颈在 n 的立方。n=800 时约 5e8，C++ 需常数优化，Python 易 TLE。

**二维表打印调试**：`n≤6` 时在 `length` 每层打印 `dp` 子矩阵，对照手算 18000。石子合并打印 `seg(i,j)` 验证前缀和。

## Python 实现

Study 文件 `interval_dp.py` 实现矩阵链乘法及自测。完整源码如下（与仓库一致）：

```python
"""区间 DP：矩阵链乘最小乘法次数。"""

from __future__ import annotations


def matrix_chain_order(p: list[int]) -> int:
    """p 长度 n+1，矩阵 i 维度 p[i] x p[i+1]；返回最少标量乘法次数。"""
    n = len(p) - 1
    if n <= 0:
        return 0
    dp = [[0] * n for _ in range(n)]
    for length in range(2, n + 1):
        for i in range(n - length + 1):
            j = i + length - 1
            dp[i][j] = 10**18
            for k in range(i, j):
                cost = dp[i][k] + dp[k + 1][j] + p[i] * p[k + 1] * p[j + 1]
                dp[i][j] = min(dp[i][j], cost)
    return dp[0][n - 1]


if __name__ == "__main__":
    assert matrix_chain_order([10, 20, 30, 40]) == 10 * 20 * 30 + 10 * 30 * 40
    assert matrix_chain_order([10]) == 0
    print("interval_dp OK")
```

**实现要点**

- `n = len(p) - 1`：矩阵个数；`p` 仅一个元素时无矩阵，返回 0。
- `length` 从 2 开始：至少两个矩阵才有合并代价。
- `dp[i][j] = 10**18`：仅对当前区间重置，单点 `dp[i][i]=0` 已初始化。

运行自测见 **Study 仓库对照**，应打印 `interval_dp OK`。

**逐行阅读 `matrix_chain_order`**

- `n = len(p) - 1`：矩阵个数；`len(p)==1` 时 `n=0` 直接返回 0。
- `dp = [[0]*n for _ in range(n)]`：单点 `dp[i][i]=0` 已满足，表示一个矩阵无需乘法。
- `for length in range(2, n+1)`：区间长度至少 2 才有合并；这是整个算法的「时间轴」。
- `i in range(n - length + 1)`：左端点范围，保证 `j=i+length-1 < n`。
- `dp[i][j] = 10**18`：当前区间先置无穷，再对 k 取 min。
- `k in range(i, j)`：切分点不能等于 j（否则右段为空矩阵链语义错误）。
- `cost = dp[i][k] + dp[k+1][j] + p[i]*p[k+1]*p[j+1]`：左右子问题 + 最后一次相乘代价。
- `return dp[0][n-1]`：整个链 `[0, n-1]` 的最优值。

**调试建议**：在 `length==2` 时打印所有 `dp[i][i+1]`，应与手算两矩阵相乘代价一致。`length==3` 时核对 `dp[0][2]` 是否为 18000。

**扩展练习（非仓库）**：实现 `matrix_chain_with_split` 返回最优括号串；实现 `stone_merge` 与矩阵链对拍随机小 n。

**与 C++ 对拍**：同一组 `p` 在 Python 与 C++ 输出应一致；Windows PowerShell 下先 `python interval_dp.py` 再 `.\run.exe`。

## C++ 实现

C++ 镜像 `interval_dp.cpp`，逻辑与 Python 一一对应：

```cpp
#include <alg_std.hpp>
#include <cassert>
using namespace std;

long long matrix_chain_order(const vector<int>& p) {
    int n = (int)p.size() - 1;
    if (n <= 0) return 0;
    vector<vector<long long>> dp(n, vector<long long>(n, 0));
    for (int len = 2; len <= n; ++len)
        for (int i = 0; i + len - 1 < n; ++i) {
            int j = i + len - 1;
            dp[i][j] = LLONG_MAX / 4;
            for (int k = i; k < j; ++k) {
                long long cost = dp[i][k] + dp[k + 1][j] + 1LL * p[i] * p[k + 1] * p[j + 1];
                dp[i][j] = min(dp[i][j], cost);
            }
        }
    return dp[0][n - 1];
}

int main() {
    assert(matrix_chain_order({10, 20, 30, 40}) == 1LL * 10 * 20 * 30 + 1LL * 10 * 30 * 40);
    assert(matrix_chain_order({10}) == 0);
    cout << "interval_dp OK" << endl;
    return 0;
}
```

**对照要点**

- `1LL * p[i] * ...` 防止 32 位溢出。
- `LLONG_MAX / 4` 与 Python `10**18` 同理，避免 `min` 时溢出。
- 循环边界 `i + len - 1 < n` 与 Python `range(n - length + 1)` 等价。

### 矩阵链维度与下标（精读）

数组 `p` 长度 `n+1` 描述 `n` 个矩阵：`Ai` 为 `p[i]×p[i+1]`。合并 `Ai..Ak` 与 `A(k+1)..Aj` 得 `p[i]×p[k+1]×p[j+1]` 次乘法。手画三个矩阵时先在纸上标 `p[0]..p[3]`，再填 `dp[0][1]`、`dp[1][2]`，最后 `dp[0][2]`。任何 `p[j]` 写成 `p[j+1]` 的笔误都会 WA。

### 石子合并 1000 与 GDOI

相邻合并模型中，最后一次合并 `[i,j]` 的代价必须是 **整段石子总和** `seg(i,j)`，因为最后一步把左右两堆合成一堆时，两堆总重就是整段和。只加 `seg(i,k)+seg(k+1,j)` 而漏 `seg(i,j)` 是经典错误。环形版本在 OI 常见，断点枚举或复制数组。

### 戳气球 312 与矩阵链对照表

| 项目 | 矩阵链 | 戳气球 |
|------|--------|--------|
| 区间 | 闭区间 [i,j] | 开区间 (i,j) |
| 切分语义 | 最后乘在一起 | 最后戳 k |
| 代价 | p[i]*p[k+1]*p[j+1] | nums[i]*nums[k]*nums[j] |
| 边界 | 单矩阵 dp[i][i]=0 | 虚拟 1 |

### 516 回文子序列（缩边型区间）

`dp[i][j]` 依赖 `dp[i+1][j-1]` 或 `dp[i+1][j]`、`dp[i][j-1]`，按长度扩展但**不枚举切分点 k 合并**，与矩阵链型不同。识别题面：若转移是「去掉两端字符」，用缩边；若「合并两段」，用切分。

### 记忆化与迭代对拍

`@lru_cache def f(i,j): return min(f(i,k)+f(k+1,j)+cost for k in range(i,j))`。`n>400` 递归可能栈溢出。小 n 对拍迭代与记忆化结果一致即可放心默写迭代版。

### 输出括号方案（Follow-up 实现思路）

`split[i][j]=argmin_k`。递归 `print(i,j)`：若 `i==j` 输出 `Ai`；否则 `print(i,k)` + `print(k+1,j)`。复杂度 O(n) 输出，空间 O(n²) 存 split。

### 四边形不等式（竞赛口述）

石子合并满足 `opt[i][j-1] <= opt[i][j] <= opt[i][j+1]` 时，内层 k 可单调指针优化到均摊 O(1)，总 O(n²)。面试答「石子可优化，矩阵链一般 O(n³)」即可。

### 与 Catalan、暴力复杂度

`n` 个矩阵括号化方案数 Catalan(n)。暴力递归树指数级。DP 每个子区间只算一次，多项式 O(n³)。理解「重叠子区间」是降复杂度的关键。

### 面试白板 30 秒版

设虚拟边界 `nums[-1]=nums[n]=1`。`dp[i][j]` 为开区间 `(i,j)` 内最后戳 `k` 的最大分：`dp[i][j]=max_{i<k<j}(dp[i][k]+dp[k][j]+nums[i]*nums[k]*nums[j])`。边界 `j=i+1` 时为 0。与矩阵链区别：**开区间**、代价为乘积。

### 面试话术

「这是区间 DP，状态是区间 `[i,j]`，按区间长度从小到大填表。枚举切分点 `k`，合并左右子区间最优再加跨越代价。矩阵链代价是 `p[i]*p[k+1]*p[j+1]`，石子合并是区间总和。复杂度 O(n³)。」

### 循环顺序归纳

`dp[i][j]` 依赖更短的子区间，故必须先算较短 `length` 再算较长。若 `i,j` 顺序随意，可能读到未计算状态导致 WA。

### 与背包、线性的边界

「重量上限、每件一次」用 `algo-dp-knapsack`；「前缀配对」可能是 LCS。连续段合并、括号化、戳破顺序用本页。312、1000 是 Hot 100 区间代表题。

### 四边形不等式（了解）

石子合并最优切分点单调，可优化到 O(n²)，竞赛向。面试可提一句，默认 O(n³)。

### 常见 Follow-up

输出括号方案：记录 `split[i][j]=k` 递归打印。环形石子：复制数组或枚举断点。代价非加性需重推转移。

### 识别速查

连续子数组合并 → 区间 DP。维度链 `p` → 矩阵链。戳破/最后操作 → 枚举最后事件。

### 面试话术

「这是 矩阵链，每件最多一次。定义 `dp[x]` 为区间 x 的最大价值，枚举矩阵时对区间长度升序更新，避免一件用多次。复杂度 O(nn)。若矩阵可无限用，区间枚举切分点。若可合并，不用 DP，用分治价值密度，见分治专题。」

### 矩阵链 长度升序的正确性（归纳）

处理矩阵 `i` 时，更新后的 `dp[x]` 应只考虑矩阵 `1..i`。更新使用 `dp[x-w]`，若内层枚举切分点，则 `dp[x-w]` 可能已在本轮放入矩阵 `i`，等价矩阵 `i` 用多次。长度升序时，所有 `x' > x` 已更新，而 `x-w < x` 尚未用当前矩阵更新，故 `dp[x-w]` 仍表示「前 i-1 件」的最优。石子合并枚举切分点则允许 `dp[x-w]` 已含当前矩阵，即重复选取。

### 与打家劫舍的类比

打家劫舍：`dp[i]=max(dp[i-1], dp[i-2]+nums[i])`，相邻不能同抢。矩阵链：`dp[x]=max(dp[x], dp[x-w]+v)`，「冲突」是区间而非下标。两者都是一维滚动，但区间的「维度」是资源消耗， robbery 的维度是时间下标。

### 常见 Follow-up

- **输出具体选了哪些矩阵？** 另开 `choice[x]` 或从 `dp` 回溯：若 `dp[x]==dp[x-w]+v` 则选了该件。
- **价值最大且重量最小？** 双关键字 DP 或第二维记录重量。
- **多区间？** 多维区间或枚举分配。
- **cap 超大？** 换价值 DP 或 meet-in-the-middle，见变体节。

### 二维费用区间（312 思路）

`dp[i][j]` 表示用了 `i` 个 0、`j` 个 1 的最大字符串个数（或能否达到）。对每个字符串 `(a,b)` 代价 `(a,b)`，矩阵链 更新：长度升序 `i,j` 双重循环，`dp[i][j]=max(dp[i][j], dp[i-a][j-b]+1)`。是 矩阵链在二维区间上的直接推广，Study 本页未实现，但循环方向仍为「费用从大到小」。

### 排列型石子合并（377 区别）

377 求石子组合**排列数**（顺序不同算不同），外层枚举区间、内层枚举石子的「石子合并」统计的是**组合**（顺序无关）。排列型需外层枚举矩阵、内层枚举切分点区间，或定义 `dp[x]` 时先遍历石子再更新。刷 516 后再做 377，避免混模板。

### 正确性一句话

矩阵链 长度升序：归纳证明处理矩阵 `i` 后 `dp[x]` 仅含前 `i` 件。完全枚举切分点：允许 `dp[x-w]` 已含矩阵 `i` 多次，归纳证明 `dp[x]` 为允许无限件的最优。分数区间分治：交换论证，与 DP 无关。

### 矩阵链 与完全对照实验

建议本地打印处理单件矩阵 `(w=2,v=10)`、`p 样例` 时两种顺序的 `dp` 数组：

- **长度升序 矩阵链**：处理前 `[0,0,0,0,0]`，处理后 `[0,10,10,10,10]`——每个区间至多吸收一次该矩阵。
- **错误枚举切分点 矩阵链**：可能得到 `[0,10,20,20,20]`，`dp[4]` 被同一矩阵更新两次，等价完全。
- **枚举切分点完全**：允许 `dp[4]=20`，符合无限件语义。

亲眼对比一次，胜过背诵十遍口诀。石子合并 Study 样例 `n=4` 建议在 `stone_merge` 末尾临时打印 `dp` 数组，对照 18000 的组成。

### 回溯输出所选矩阵（矩阵链）

在 `matrix_chain_order` 外维护 `choice[x]`：当 `dp[x]` 由 `dp[x-w]+v` 改进时记录矩阵 id。从 `x=cap` 开始，若 `choice[x]!=-1`，则选中该矩阵并令 `x -= w[id]`，否则 `x` 不变或表示未使用（依实现而定）。回溯复杂度 `O(cap)`。面试 Follow-up 常考，需在二维版理解后再压到一维。

### 幂次与二进制拆分补充说明

多重区间将件数 `c` 拆为 `1,2,4,...,2^k` 与余数，是因为二进制表示可覆盖 `0..c` 任意选取件数；拆完每件变成 矩阵链 矩阵后，总矩阵数约 `O(n log C)`，再对区间长度升序。竞赛若出现「每种最多三件」，也可直接展开三件 矩阵链，面试小数据可行。理解拆分有助于解释「为何多重最终仍用 矩阵链 长度升序」——本质没有新的循环方向，只是矩阵列表变长。

### 最后提醒

写区间题时，**先写方向再写循环体**；许多 nA 不是转移式错，而是 `for x in range(cap, w-1, -1)` 写成了 `range(w, cap+1)`。把这两行在模板中用注释标出「矩阵链 长度升序 / 完全枚举切分点」，可显著降低失误率。与 `algo-dp-linear` 对照时，重点记住：可合并 → 排序分治；不可合并 → 本页 DP。两条规则覆盖绝大多数区间类题面表述。
