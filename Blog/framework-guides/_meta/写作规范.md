# Framework 系列 · 官方指南体例规范

## 定位

对标 [Vue 官方指南](https://cn.vuejs.org/guide/introduction.html)：**单页即一整套教程**——导读、快速上手、与官方 Tutorial 对齐的**完整基础篇**、Framework 子工程实战。读者无需跳转外站即可学完该栈基础知识，再落到本仓示例验证。

体例标杆：[beego-go/index.md](../beego-go/index.md) — 概念说明 + 可运行示例 + TIP/注意；子工程对照**写在叙述中**，**不是**独立「Framework 对照」段或附录。

**独立成文**：单篇指南须能单独阅读完成，**不得**要求读者先读其它 slug 的指南，**不得**用「（再读）」重复串联它篇主题，**不得**以「与同仓 XXX 栈对照」替代对本栈的讲解（仅可对照**本子工程**源码与栈官方文档）。

## 单页五大块（顺序固定，缺一则未达标）

1. **导读** — 什么是 {Stack}、适用场景、**最小 Hello 示例**、核心能力 2～4 条  
2. **预备知识** — 用引用块 `>` 标出前置技能与版本  
3. **快速上手** — 安装、进入/创建工程、启动、首次验证（Windows PowerShell）  
4. **基础篇** — 章节标题必须与 `Blog/framework-guides/_meta/guide-toc/{guide_toc}.yaml` 中 `essentials` 列表一致  
5. **Framework 子工程实战** — 目录树、跑通步骤、关键源码、health/info 或该栈约定接口  

另含：**学习路径**、**延伸阅读**（官方文档链接）。

## 基础篇每节写法

对标 Vue 官方指南：**文字说明 → 代码 → 文字说明 → 代码**，在同一节内交替展开；不要用「附录」「常见问题大全」「实验手册」等独立大块重复正文。

- 2～4 段概念（禁止只写「详见官方」）  
- ≥1 个完整代码块（优先独立可运行示例）  
- 可选 `> **TIP**` / `> **注意**`  
- 子工程相关说明写在叙述中自然带出，**不要**单独开「附录：逐段导读」  

## 篇幅与 tier

| guide_tier | 全文汉字 | 说明 |
|------------|----------|------|
| `major` | ≥ 25000 | FastAPI、NestJS、React、Next、Django、Flask 等 |
| `medium` | ≥ 12000 | 常规栈 |
| `placeholder` | ≥ 6000 | HTTP/CLI 占位、形态说明 |

发布：`status: published` **仅当** `validate_guide.py` 与 `validate_guide_quality.py` 均 `--strict` 通过，且达到上表全量汉字门槛（`guide_lib.TIER_MIN_CHARS`，无样板豁免）。

允许的顶层 `##`（顺序固定）：导读、预备知识、快速上手、基础篇、Framework 子工程实战、学习路径、延伸阅读。**禁止**附录、常见问题大全、PowerShell 实验步骤、源码阅读索引等额外 `##` 大块。

## 维护与撰写

正文须 **逐篇原创撰写** 或人工精修，**不得**用 `scripts/archive/legacy/` 下的批量生成脚本覆盖 `index.md`。

**允许使用的维护脚本**（在 atelier 根目录执行）：

| 脚本 | 用途 |
|------|------|
| `scripts/extract_framework_sources.py` | 从 Study 抽取素材 |
| `scripts/generate_guide_skeleton.py` | 仅生成骨架，不代替正文 |
| `scripts/validate_guide.py` | 结构 / tier 字数 |
| `scripts/validate_guide_quality.py` | 灌水、错栈 lint |
| `scripts/publish_valid_guides.py` | 校验通过后改 manifest `published` |
| `scripts/assign_guide_toc.py` | manifest `guide_toc` |
| `scripts/gen_manual_progress.py` | 更新进度表 |
| `scripts/manual_wave_map.py` | 波次 slug 映射 |
| `scripts/scan_framework_docs.py` | 扫描 Study 文档 |
| `scripts/migrate_manifest_paths.py` | manifest 路径字段迁移（一次性） |

站点资源：`scripts/site/fetch_cert_images.py`、`scripts/site/fetch_wiki_textures.py`。

进度登记：`Blog/framework-guides/_meta/人工撰写进度.md`。

精写阶段每篇须对照 `{FRAMEWORK_ROOT}/{repo_path}` **人工重写**（默认 `F:\Study\Framework`，可在 `config/site.local.json` 或环境变量 `FRAMEWORK_ROOT` 覆盖）。

## 禁止

- 旧 Wiki 八段标题（摘要、技术定位…）  
- 配图 `images/`  
- 用固定句段重复粘贴凑字数（如「学习时建议：先手写最小示例…」整段在多节复用）  
- 在 `## 基础篇` 下使用 `#### 学习要点` 等四级标题堆砌通用段  
- 非 Python 栈正文出现 `uvicorn`、`Pydantic`、`Depends(` 等其它栈专属术语  
- 抄袭官方文档原文（目录可对齐，正文须原创中文）

## 标题

- frontmatter `title`：`{Stack} 官方指南：从入门到 Framework 子工程实战`  
- 文首 **目录** 锚点覆盖所有 `##` 基础篇章节  

## 素材

- `{FRAMEWORK_ROOT}/{repo_path}` + 栈指南 `*.md`  
- `scripts/extract_framework_sources.py --slug xxx`  
- `scripts/generate_guide_skeleton.py` 仅生成骨架，不代替正文
