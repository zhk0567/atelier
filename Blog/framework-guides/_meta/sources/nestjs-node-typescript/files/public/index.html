本示例体现的 Nest 特点

  • ConfigModule：全局加载环境变量（可扩展为配置服务）。
  • 模块边界：`Health` / `Demo` / `Items` / `FeatureBox` 分模块注册。
  • 依赖注入：`ItemsService` 注入到 `ItemsController`。
  • 管道：`ValidationPipe` + `class-validator` DTO。
  • 拦截器分层：全局 `LoggingInterceptor` + 路由级演示拦截器。
  • Swagger：OpenAPI 与 UI 在 /docs。
